1) What this project simulates
==============================
Luminis-MC is a Monte Carlo simulator of *polarized* light transport in
random scattering media, with full Jones-vector tracking and a reciprocity-based
model of coherent backscattering (CBS).

Primary scientific goal:
  - Study the CBS enhancement cone as the interference between each
    multiple-scattering path and its exact time-reversed (reciprocal) partner.
    The observable is the angular enhancement factor
        eta(theta) = S0_coherent(theta) / S0_incoherent(theta),
    which tends to 2 at exact backscatter (theta -> 0) for a reciprocal,
    conservative medium and falls to 1 in the diffuse wings.

Secondary capabilities:
  - Time-resolved detection (photon time-of-flight gating).
  - Layered (stratified) samples sharing one host refractive index, where each
    layer is either a single species (HomogeneousLayer) or a mixture of several
    co-located species chosen per scattering event (MixtureLayer, §4.9b).
  - Absorption deposition maps on a cylindrical (r,z) grid.
  - Statistics histograms (events / theta / phi / depth / time / weight).
  - Speckle / near-field complex-field accumulation.
  - HDF5 persistence with a typed, schema-driven layer for large campaigns.

=============================
Build system: CMake + scikit-build-core; C++23; Fortran for Mie.

Targets (CMakeLists.txt):
  mie_fortran  (STATIC)  : src/mie/MIEV0noP.f + ErrPack.f + dmiev.cpp.
                           GNU Fortran is forced to 8-byte reals/doubles
                           (-fdefault-real-8 -fdefault-double-8).
  _core_headers (INTERFACE): the include/ tree.
  _core_c      (STATIC)  : the engine .cpp list (absortion, core, detector,
                           laser, meanfreepath, medium, phase, photon, results,
                           rng, sample, simulation, table, vec). Links mie_fortran.
  _core        (pybind11 module): python/python_module.cpp, links _core_c.

Build-type / optimization knobs:
  - Default build type is **Release** (forced when none is given). The
    per-photon kernel MUST be compiled -O3 -DNDEBUG; a Debug build runs the hot
    loop unoptimized and Matrix/CMatrix operator() asserts stay live.
  - LUMINIS_NATIVE=ON  -> -march=native (vectorize for the host CPU; the .so is
                          then non-portable across microarchitectures).
                          Still IEEE-strict: there is intentionally NO
                          -ffast-math, so the physics stays bit-reproducible.
  - LUMINIS_LTO=ON     -> interprocedural/link-time optimization (if supported).

  Standalone optimized build:
    cmake -S . -B build -DCMAKE_BUILD_TYPE=Release \
          -DLUMINIS_NATIVE=ON -DLUMINIS_LTO=ON
  pyproject.toml pins build-type=Release and LUMINIS_NATIVE/LUMINIS_LTO=ON, so a
  plain `pip install .` already produces an optimized native build.

Installed Python package: luminis_mc (the _core extension + .pyi stub + py.typed).


4) Physical model and assumptions
=================================
The engine is a Monte Carlo solver of the polarized radiative-transfer picture,
with a coherent pair-interference layer added on top for CBS observables.

4.1 Transport picture
  At the scalar level the packet obeys the radiative-transfer form
    dI/ds = -mu_t I + mu_s ∫ p(s'->s) I dΩ' + S,   mu_t = mu_a + mu_s.
  The Monte Carlo realization:
    - sample a free path stochastically,
    - scatter the direction with a phase-law sampler,
    - carry the polarization as a Jones vector with an amplitude matrix,
    - track absorption as packet-weight loss (variance reduction),
    - let detector estimators accumulate the observables.

4.2 Free-path sampling
  Exponential in physical length using the medium's mean_free_path l = 1/mu_s:
    l = -mean_free_path * ln(U),   U ~ Uniform(0,1).
  IMPORTANT: both RGDMedium and MieMedium sample from the `mean_free_path`
  FIELD, NOT from mu_s. You must call set_mean_free_path(...) or free paths come
  out 0 and the photons never move. mu_s/mu_a/mu_t (set via
  set_scattering_coefficient/set_absorption_coefficient) drive the absorption
  split and the estimator/optical-depth attenuation, which is a *separate* knob
  from the geometric step length.

4.3 Two path-length accumulators (do not confuse them)
  Each Photon carries two distinct path sums:
    - opticalpath   = Σ d_i           GEOMETRIC distance. Used for time of
                                      flight: t_arrival = launch_time +
                                      opticalpath / velocity. (The name is kept
                                      for API stability; it is physical distance,
                                      not Σ n·d.)
    - optical_path  = Σ n_i·d_i       OPTICAL distance. Used for the propagation
                                      phase exp(i·k·optical_path) with k the
                                      free-space wave number.
  With a single shared host index these differ only by the constant factor n,
  but they are kept separate so a future per-layer n can be introduced without
  touching the detectors.

4.4 Polarization state and the moving local frame
  Each photon stores a 3x3 frame matrix P_local whose ROWS are
      row 0 = m  (transverse "horizontal"),
      row 1 = n  (transverse "vertical"),
      row 2 = s  (propagation direction).
  The Jones vector is E = (E_m, E_n) in the (m,n) basis. At each event the frame
  rotates by the 3x3 matrix A(theta,phi) (built in run_photon, simulation.cpp):
      P_local(new) = A(theta,phi) · P_local(old),  rows = (m', n', s').
    A(0,*) = ( cosθ cosφ,  cosθ sinφ, -sinθ )
    A(1,*) = (     -sinφ,       cosφ,      0 )
    A(2,*) = ( sinθ cosφ,  sinθ sinφ,  cosθ )

4.5 Scattering operator and amplitude normalization
  The medium returns the 2x2 amplitude scattering matrix S(θ) in the (p,s) basis,
  diagonal by construction:
      S = | s2  0 |   (s2 acts on the parallel/p component,
          |  0 s1 |    s1 on the perpendicular/s component).
  The per-event Jones operator is T = S(θ)·R(φ), with the in-plane rotation
      R(φ) = |  cosφ  sinφ |
             | -sinφ  cosφ |.
  The Jones vector is updated with an amplitude normalization:
      E_out = T·E_in / sqrt(F),
  where F is the expected scattered intensity for the current (E_in, φ):
      F = |E_m|² (|s2|² cos²φ + |s1|² sin²φ)
        + |E_n|² (|s2|² sin²φ + |s1|² cos²φ)
        + 2 Re(E_m E_n*) (|s2|² - |s1|²) sinφ cosφ.
  Dividing by sqrt(F) keeps |E| ~ 1; the energy/intensity bookkeeping lives in
  photon.weight instead. This is exactly the same F used by the estimator
  (phase_F in detector.cpp) and by the conditional-φ sampler, so all three stay
  consistent.

4.6 Conditional (polarization-aware) azimuth sampling
  φ is NOT sampled uniformly when the photon is polarized. The conditional
  sampler (PhaseFunction::sample_phi_conditional, phase.cpp) draws φ ∝ F(φ):
      F(φ) = a cos²φ + b sin²φ + c sinφ cosφ,
      a = |s2|²|E_m|² + |s1|²|E_n|²,
      b = |s1|²|E_m|² + |s2|²|E_n|²,
      c = 2 (|s2|² - |s1|²) Re(E_m E_n*),
  by rejection against the closed-form maximum
      Fmax = ½(a+b) + ½ sqrt((a-b)² + c²).
  This correctly couples the azimuth to the polarization state, which is what
  produces the polarization structure of the CBS cone.

4.7 Medium models
  RGDMedium (Rayleigh-Gans-Debye): valid for small, optically soft particles
  (size parameter x = 2π r/λ ≪ 1, |m-1| ≪ 1). Amplitudes:
      s2 = -i k³ (m-1) V F(θ) cosθ / (2π),
      s1 = -i k³ (m-1) V F(θ)        / (2π),   V = (4/3)π r³,  m = n_p/n_med.
  The constant prefactor -k³(m-1)V/(2π) is precomputed in the ctor (rgd_prefactor),
  and — like Mie — the whole s1(θ)/s2(θ) curve is TABULATED on a uniform θ grid
  (2000 nodes) at construction (precompute_scattering_tables). This turns each
  scattering_matrix() call into two O(1) DataTable look-ups + a lerp instead of
  recomputing the form factor (sin/cos/pow) — critical because the CBS estimator
  hits scattering_matrix hundreds of times per event.

  MieMedium: full Mie theory via the MIEV0 Fortran solver. S1(θ),S2(θ) are
  precomputed over a uniform θ grid (1000 nodes) in the ctor and stored as
  DataTables for O(1) interpolation. The grid is built directly in θ and stored
  uniform (no acos(cosθ) round-trip), enabling direct-index look-ups.

  Form factor (phase.cpp, also exposed to Python as form_factor):
      F(θ) = 3(sin u - u cos u)/u³,  u = 2 k r sin(θ/2),  F(0)=1.

4.8 Phase functions (angular θ sampling) vs amplitude matrices
  Two separate concepts share the angle θ:
    - PhaseFunction (phase.hpp): builds an inverse-CDF SamplingTable from a PDF
      and samples θ (sample_theta) and φ. Concrete types: Uniform, Rayleigh,
      HenyeyGreenstein, RayleighDebye, RayleighDebyeEMC, Draine, Mie. The EMC
      and Mie variants also report scattering_efficiency() (Q_sca) and
      scattering_cross_section() (Q_sca·π r²); Mie's Q_sca comes from mievinfo,
      EMC's from a Simpson integration of F²(θ)·(1+cos²θ).
    - ScatteringMedium::scattering_matrix: the 2x2 amplitude matrix S(θ) used
      for the polarization transform and for CBS.
  The medium owns BOTH: the phase function samples the angle; the amplitude
  matrix transforms the field. Mind that the EMC/Mie phase functions and the
  RGD/Mie media are configured independently and should be kept physically
  consistent (same wavelength/radius/indices).

4.9 Layered sample and interface crossing
  A Sample is an ordered, contiguous stack of `Layer` slabs along z, sharing
  ONE host refractive index n_medium (Sample.refractive_index) and the derived
  light_speed = 1/n_medium (natural units). `Layer` is an ABSTRACT base holding
  only the z-geometry ([z_min, z_max), contains, thickness) plus the aggregate
  transport interface the kernel consumes (sample_free_path, mu_attenuation,
  mu_scattering, mu_absorption, select_scatter_medium). Two concrete types:
    - HomogeneousLayer: one co-located ScatteringMedium (the historical
      one-species-per-depth behaviour). Every transport accessor delegates to the
      wrapped medium, and select_scatter_medium returns it WITHOUT drawing from
      the RNG — so single-species runs stay bit-for-bit reproducible.
    - MixtureLayer: several co-located species, one chosen per event (§4.9b).
  Sample owns its layers polymorphically as `std::vector<unique_ptr<Layer>>`, so
  Sample is MOVE-ONLY (copy is explicitly deleted; see the sample.hpp comment on
  why libc++ would otherwise let pybind11 pick an ill-formed copy path).
  Consequences of the shared host index:
    - Internal interfaces change only (mu_a, mu_s, phase model / amplitude
      matrix); the refractive index is uniform, so there is NO internal Fresnel
      reflection/refraction.
    - When a sampled step crosses an interface after a sub-length l1, the
      remaining OPTICAL depth is preserved:
        tau_remaining = (l - l1) * mu_t(old);  l_new = tau_remaining / mu_t(new).
      mu_t here is the LAYER'S aggregate mu_attenuation() (for a mixture, the sum
      over species — not any single one).
    - After each crossing, photon.z is nudged by ±1e-12 so the layer lookup is
      robust against floating-point boundary ambiguity.
    - Sensor crossings are tested on every sub-step (so a detector between the
      start and the interface still fires).

4.9b Mixture layers (several co-located species)
  A MixtureLayer models a slab in which several particle species coexist over the
  SAME z-range. It assumes the dilute / independent-scattering regime: no
  structure factor, the species simply ADD their scattering coefficients. Each
  species i contributes
      mu_s^(i) = n_i · sigma_s^(i),
  the product of its number density n_i [1/mm³] and its single-particle
  scattering cross-section sigma_s^(i) [mm²]. sigma_s^(i) comes from the species'
  phase function (ScatteringMedium::scattering_cross_section() delegates to it;
  EMC/Mie return a real value, other phase functions return 0). All aggregates are
  precomputed once in the ctor:
      mu_s_total = Σ mu_s^(i),
      mu_a_total = Σ mu_s^(i)·(mu_a/mu_s)_i    (preserves each species' single-
                                               scattering albedo; 0 if mu_s^(i)=0),
      mu_t_total = mu_s_total + mu_a_total,
      mfp_total  = 1 / mu_s_total.
  Two independent random decisions per vertex, in this order:
    1. FREE PATH: sample_free_path draws an exponential with mfp_total (the
       aggregate mean free path of the whole mixture).
    2. SPECIES: select_scatter_medium draws ONE species j with probability
       p_j = mu_s^(j) / mu_s_total via a lower_bound on a normalized selection CDF
       (the CDF's last entry is pinned to 1 to guard round-off).
  The chosen species drives S(θ), F and the angular normalization at that vertex;
  the LAYER's aggregate albedo (mu_s_total/mu_t_total) drives the absorption split.
  For a HomogeneousLayer the two coincide, so the single-species path is
  unchanged. The MixtureLayer ctor validates its inputs and throws
  std::invalid_argument on empty/mismatched lists, a null species, or
  mu_s_total ≤ 0 (e.g. every species' phase function reports a zero cross-section).

4.10 Absorption split and Russian roulette
  At each scattering event, with the layer where the scatter occurred:
      d_weight = w * (mu_a / mu_t);   w <- w * (mu_s / mu_t).
  d_weight is deposited into the Absorption grid (if enabled). Low-weight
  packets (w < 1e-4) undergo Russian roulette: survive with p = 0.1 and rescale
  w <- w/0.1, else terminate. A hard cap MAX_EVENTS (default 1000) also bounds
  the path length.

4.11 Time of flight and units
  t_arrival = launch_time + opticalpath_at_hit / velocity, velocity = c/n in
  natural units (= light_speed = 1/n_medium). The laser temporal parameters are
  given in picoseconds and converted to nanoseconds internally (PS_TO_NS = 1e-3;
  Gaussian width uses FWHM->sigma). launch_time is stored in ns.

4.12 Stokes observables
  From lab-frame field components (Ex,Ey):
      S0 = |Ex|²+|Ey|²,  S1 = |Ex|²-|Ey|²,
      S2 = 2 Re(Ex Ey*), S3 = 2 Im(Ex Ey*).
  Photon::get_stokes_parameters uses the same convention in the local (m,n)
  basis but with S3 = -2 Im(E_m E_n*) (RHC-positive sign convention). Planar and
  far-field grids are Monte Carlo sums of these per-contribution.

4.13 CBS cone-width interpretation (post-processing only)
  In diffusion-like analysis, the reduced variable q = k l* θ gives a cone of
  FWHM Δq ≈ 0.73, i.e. θ_FWHM ≈ 0.73/(k l*), with l* the transport mean free
  path. derived_quantities() in manager.py centralizes l_s, l* = l_s/(1-g) and
  theta_coherent = 1/(k l*). For a mixture, derived_quantities_mixture(species,
  number_densities) does the same from the per-species mu_s^(i) and g_i:
  effective anisotropy g_eff = Σ mu_s^(i) g_i / Σ mu_s^(i), l_s = 1/Σ mu_s^(i),
  and l* = 1/Σ mu_s^(i)(1-g_i). This is an interpretation model, not a constraint
  inside the engine.


5) Core C++ types
=================
Photon (photon.hpp): the live packet. Position, P_local frame, Jones vector
  (polarization), weight, two path sums (opticalpath/optical_path), event count,
  current_layer, launch_time, velocity, k, penetration_depth, polarized flag,
  and the CBS bookkeeping (next section).
  Dead/reserved fields (kept for ABI/clarity, NOT used): detected_pos (never
  written) and coherent_path_calculated (only ever reset to false, never read).
  matrix_T accumulates ONLY the interior normalized Jones products J_2..J_{n-1}
  (the first and last events are applied separately in reverse_field), not "all"
  matrices.
  Per-vertex species pointers (non-owning, for mixtures): active_medium is the
  species selected for the CURRENT event; first_scatter_medium and
  last_scatter_medium record the species at the first (r_1) and last (r_n) vertex
  for CBS Stages C and A. All three are reset to nullptr when a photon is reused.
  For a HomogeneousLayer they always equal that layer's single medium.

PhotonRecord (photon.hpp): immutable detection snapshot written by
  PhotonRecordSensor (events, depth, times, optical path, weight, k, first/last
  scatter positions, detector position, direction + (m,n) frame, forward and
  reverse Jones vectors).

Sample / Layer / HomogeneousLayer / MixtureLayer (sample.hpp): see §4.9–4.9b.
  `Layer` is the abstract base (z-geometry + aggregate transport interface);
  HomogeneousLayer wraps one medium, MixtureLayer holds several species.
  add_layer(medium, ...) and add_mixture_layer(species, number_densities, ...)
  both enforce contiguity (each layer starts where the last ended; first at z=0;
  top may be +∞) via the shared validate_contiguous helper. Layers are owned as
  unique_ptr<Layer>, so Sample is move-only. get_layer returns `const Layer&`;
  get_layer_at / get_layer_index_at (O(log N) binary search on interfaces),
  find_next_interface, is_inside, z_top, light_speed_in_medium.

ScatteringMedium / RGDMedium / MieMedium (medium.hpp): see §4.7. Constructors do
  NOT set optical coefficients — you set mu_s/mu_a explicitly, and
  mean_free_path separately.

Laser (laser.hpp): source geometry (Point / Uniform disk / Gaussian, radius or
  std-dev = sigma) and temporal profile (Delta/Gaussian/TopHat/Exponential/
  PulseTrain/CW). emit_photon samples the launch position + emission time and
  sets the initial Jones state. Default beam: position (0,0,1e-9), direction +z,
  m=+x, n=+y.

SimConfig (simulation.hpp): non-owning run config. n_photons, n_threads (0 ->
  hardware_concurrency), seed, batch_size (0 -> 256), MAX_EVENTS, show_progress,
  progress_interval_pct, pin_threads_to_cores, track_reverse_paths, and pointers
  to sample/laser/detector (required) and absorption (optional).


6) The transport kernel (run_photon, simulation.cpp)
====================================================
Per-photon loop (while alive):
  0. (Once per photon) Reset CBS history to the launch state: r_1=r_n=pos;
     P0=P1=Pn2=Pn1=Pn=P_local; initial_polarization=polarization;
     matrix_T=identity; has_T_prev=false. Per-event scratch matrices (A, R,
     T_current, the matrix_T products, S_matrix) are allocated ONCE here and
     overwritten in place — the hot loop does zero heap allocations per event.
  1. Resolve current layer/medium; sample a free path.
  2. Interface handling: while the step crosses a boundary, truncate at the
     interface, preserve remaining optical depth, nudge ±1e-12 into the next
     layer, test sensor crossings on the sub-step, and continue.
  3. Advance the remaining step in the final layer; update both path sums and
     penetration_depth.
  4. Test detector crossings (record_hit); kill if absorbed.
  5. Boundary check (is_inside); kill if escaped.
  6. Select the scattering species for this vertex: photon.active_medium =
     layer.select_scatter_medium(rng) (a HomogeneousLayer draws nothing; a
     MixtureLayer draws one species ∝ mu_s^(i)). Store it on the photon FIRST so
     the estimator and the real scatter below use the SAME species at this vertex,
     then run detector.run_estimators(photon, sample).
  7. Sample θ (from active_medium's phase function), evaluate S(θ), then sample φ
     conditionally.
  8. Update P_local via A(θ,φ).
  9. If polarized: build T=S·R, compute F, update the Jones vector by T/√F.
 10. If track_reverse_paths: update the CBS frame snapshots and the interior
     T-matrix (next section).
 11. events++, absorption split + record, Russian roulette, MAX_EVENTS cap.

Parallel driver (run_simulation_parallel):
  - Clone-merge: each worker gets its own SensorsGroup clone (zeroed
    accumulators) and Absorption clone; results are merged additively
    (merge_from, matched by sensor id) after join.
  - Dynamic batch scheduling: workers grab [start,start+batch) ranges via an
    atomic fetch_add for load balance.
  - REPRODUCIBILITY: the RNG (xoshiro256++, seeded by SplitMix64) is RE-SEEDED
    PER PHOTON INDEX with mix_seed(seed, i) at the top of each iteration. So a
    given photon always draws the same stream regardless of which thread grabs
    it — the output is bit-identical for a fixed seed across runs AND across
    thread counts. (Double-precision grids summed in different orders can still
    differ at ~1e-15 from merge ordering; this is float summation noise, not a
    regression, and exact bit-equality of summed grids is not attainable.)
  - The first exception thrown in any worker is captured under a mutex and
    re-thrown after all threads join.
  - macOS pinning is only a scheduler affinity hint; Linux uses real
    pthread_setaffinity_np. Pinning is best-effort and off by default.


7) Sensor system
================
All sensors are infinite z-planes (normal +z). Two detection modes:
  - Direct hit: the photon physically crosses the plane. SensorsGroup::record_hit
    finds crossed planes via a sorted z_layers map, computes the exact
    ray-plane intersection, corrects the path length for the overshoot beyond
    the plane (both phase via optical_path and time via opticalpath), builds an
    InteractionInfo {intersection_point, phase=exp(i k optical_path_corr),
    crossing_direction, opticalpath_at_hit}, applies the filters, and calls
    process_hit.
  - Estimator (next-event / last-flight): after EVERY scattering event,
    run_estimators calls process_estimation on estimator-enabled sensors,
    accumulating a virtual contribution toward the detector without requiring a
    physical crossing (variance reduction).

  IMPORTANT: a sensor is EITHER direct OR estimator. record_hit skips
  estimator-enabled sensors entirely (they would double-count and inject
  high-variance single-bin spikes on top of the smooth estimator curve). The
  estimator path is only physically valid at z=0; the Sensor ctor disables
  estimator mode with a warning if z≠0.

Filters (Sensor::check_conditions, conjunctive, short-circuiting): crossing
  direction, event-count range (set_events_limit, inclusive), polar angle theta
  (cached cos bounds; theta = acos(-dir_z)), azimuth phi (atan2 in [0,2π)),
  and rectangular (x,y) window. The events filter also gates the CBS estimator
  (it bypasses check_conditions, so process_estimation re-checks it manually;
  note the estimated path has events+1 scatters, so to match direct order N set
  the limit to N-1).

Sensor types:
  1. PhotonRecordSensor — one PhotonRecord per hit. Memory grows with hit count.
  2. PlanarFieldSensor — accumulates complex Ex/Ey (amplitudes, with phase and
     sqrt(weight)) on an (x,y) grid → speckle/coherent imaging.
     process_estimation is NOT implemented (the body is commented out).
  3. PlanarFluenceSensor — accumulates Stokes S0..S3 on an (x,y) grid, optionally
     time-resolved. process_estimation is NOT implemented.
  4. FarFieldCBSSensor — the main CBS detector (see §8). At z=0.
  5. StatisticsSensor — configurable histograms (events/theta/phi/depth/time/
     weight), optionally time-split.

  (There is no FarFieldFluenceSensor or PlanarCBSSensor in the current build/
  bindings; older docs mentioning them are stale.)

SensorsGroup owns the sensors (unique_ptr) and indexes them by z in an ordered
  map for fast crossing lookups. add_detector TAKES OWNERSHIP: the Python
  binding clones the passed sensor, stores it, and returns a reference to the
  internal copy — so after a run you read results from the returned handle /
  grp.detectors[i], not from your original Python object.


8) CBS implementation (detector.cpp)
====================================
CBS is computed entirely in detector.cpp as forward/reverse pair interference.
The live algorithm is the file-local reverse_field() (NOT "coherent_calculation",
which is a stale name; and NOT src/cbs.cpp, which is DEAD CODE — see §14).

8.1 Pair picture
  A path with ≥2 scattering events has a time-reversed partner that visits the
  same scatterers in reverse order. Near exact backscatter their phases match
  and they interfere constructively, lifting the signal above the diffuse
  background — the CBS cone. Single-scatter paths are their own reverse and
  carry no enhancement (they are deposited identically in the coherent and
  incoherent channels, i.e. they form a flat background).

8.2 Reverse-field reconstruction (reverse_field, Xu 2008 Eq. 3)
  Using the stored frame snapshots (P0,P1,Pn1,Pn), the first/last scatter SPECIES
  (photon.first_scatter_medium / last_scatter_medium — the amplitude matrices for
  Stages C and A are taken from the species the photon actually scattered against,
  NOT from the layer index, so mixtures are correct), the RAW interior product
  matrix_T_raw, and the launch polarization, the reverse Jones vector is built in
  three stages:
      E_rev = S^(1) R(φ1') · T^rev · R(φn') S^(n) R(φn) · E0,   T^rev = Q T^T Q,
      Q = diag(1,-1).
    Stage A: first reverse scatter at r_n, s0 -> -s_{n-1}.
    Stage B: interior segment via reciprocity, E <- Q · T_interior^T · Q · E.
    Stage C: last reverse scatter at r_1, -s1 -> s_n, then normalize once.
  It uses the RAW (un-F-normalized) interior product and normalizes the final
  vector exactly once; any scalar inside the path cancels because energy is
  carried by photon.weight, keeping it consistent with the normalized forward
  field. The function is split into reverse_field_prefix (Stages A+B) and
  reverse_field_suffix (Stage C): the prefix is independent of the exit
  direction, so the estimator computes it once and runs only Stage C per
  angular bin (a big speedup; the split is bit-exact vs the monolith).

8.3 Geometric phase and interference (process_hit)
  The CBS geometric pair phase is
      path_phase = exp(i (s_out + s_in)·k·(r_n - r_1)).
  At exact backscatter s_out = -s_in the phase vanishes → constructive. Forward
  and reverse lab-frame fields are combined as E_tot = E_f + path_phase·E_r.
  The coherent channel accumulates |E_tot|² Stokes; the incoherent channel
  accumulates |E_f|² + |E_r|² Stokes (no interference) as the baseline.

8.4 The interior T-matrix one-event lookahead
  Because we don't know which event is "last" until detection, matrix_T (and the
  raw matrix_T_raw) are accumulated with a one-event buffer: the most recent
  event is held as the "last" candidate; when the next event arrives, the
  buffered one is confirmed interior and folded into the product. matrix_T_raw is
  Frobenius-normalized each fold to avoid overflow on very long paths (the scale
  cancels in reverse_field).

8.5 Direct detector (FarFieldCBSSensor::process_hit)
  Far-field angular bin: theta = acos(-s_out.z) (theta=0 is exact backscatter,
  the -z direction), phi = atan2(s_out.y, s_out.x). events<2 → single-scatter
  background branch (Er=0, coherent==incoherent). events≥2 → full reverse_field +
  geometric phase. Photons are also time-binned (§9).

8.6 Estimator (FarFieldCBSSensor::process_estimation)
  Forced detection over every (theta,phi) bin after each event:
    - The estimator uses photon.active_medium (the species selected for this
      vertex) for S(θ), F and the angular normalization, but the LAYER's aggregate
      albedo (mu_scattering/mu_attenuation) for the scatter weight w_scatter.
    - I_norm = ∫(|s1|²+|s2|²) sinθ dθ is cached in a per-species map
      (_I_norm_by_medium), keyed on the medium pointer; k acts only as an
      invalidation guard (the whole map is cleared if k changes). A single-slot
      cache would recompute the integral every event in a mixture, since
      active_medium changes vertex to vertex.
    - For each bin: build s_out, intersect the z=0 plane (distance L), compute
      the layer-by-layer transmittance Tr = exp(-Σ μ_{t,i} L_i)
      (optical_depth_to_detector; fast path μ_t·L for single-layer samples),
      the scatter geometry (th_scat, sin/cos φ in the current basis), the
      angular density p(Ω) = F/(π·I_norm), and the weight into the bin
      w_bin = w_scatter · Tr · p(Ω)·(dΩ_band·dφ).
    - Forward field via apply_scatter_normalized; reverse via the hoisted prefix
      + per-bin suffix; geometric phase with the estimated r_n = current pos.
    - events==0 → single-scatter background branch (Er=0). events≥1 → full pair
      (the estimated path has events+1 ≥ 2 scatters).
  theta_pp_max caps the estimator's angular reach (defaults to theta_max if <0).
  The angular-bin trig (cos/sin of centers, per-θ solid-angle band) is
  precomputed in the ctor (and thus re-run by clone()).

  NEVER enable both modes on the same sensor instance — the angular
  distribution would be double-counted.

8.7 What is stored / validity
  For each (theta, phi, t) bin: S0_coh..S3_coh and S0_incoh..S3_incoh.
  eta(theta,phi,t) = S0_coh/S0_incoh → ~2 at the peak for an ideal reciprocal,
  conservative medium. Requirements: track_reverse_paths=true (otherwise the
  reverse-path state is not maintained) and ≥2 events for an enhanced pair. The
  model assumes reciprocal media (no magneto-optic / non-reciprocal effects).


9) Time-resolved conventions (shared across detectors and absorption)
====================================================================
One convention everywhere:
  - dt == 0 : a single bin (N_t = 1), the time-integrated signal.
  - dt  > 0 : N_t = ceil(len_t / dt) + 1. Bin 0 is ALWAYS the integrated signal;
              bins 1..N_t-1 are explicit time windows. A contribution lands in
              bin 0 and (if its arrival falls in range) in exactly one window
              bin. Arrivals outside [0, len_t) are dropped from the window bins.
Applies to PlanarFluenceSensor, FarFieldCBSSensor (FarFieldCBSSensor::time_bin),
StatisticsSensor (set_time_resolution), and Absorption.
StatisticsSensor also has a separate time HISTOGRAM (set_time_histogram_bins)
that bins arrival time directly into [1, n_bins) without an integrated bin 0.


10) Absorption recorder (absortion.hpp / absortion.cpp)
=======================================================
Cylindrical (r,z) deposition grid with optional time slices (§9 convention).
n_r = floor(radius/d_r)+1, n_z = floor(depth/d_z)+1. record_absorption deposits
d_weight at the photon's (r,z) (and time bin). time_slices[0] is the integrated
grid (there is no separate "total" matrix). get_absorption_image(n_photons,
time_index) returns a (2·n_r)×n_z image normalized by voxel volume and photon
count and mirrored along the radial axis; get_total_image = time_index 0.


11) Post-processing (results.hpp / results.cpp)
===============================================
  - postprocess_planar_fluence(det, n_photons, normalize_per_photon=True,
        normalize_per_area=True) -> PlanarFluenceProcessed {S0..S3 per time bin}.
  - postprocess_planar_field(det, n_photons, ...) -> PlanarFieldProcessed {Ex,Ey}.
  - postprocess_farfield_cbs(det, n_photons, normalize_per_solid_angle=True,
        normalize_per_photon=True) -> FarFieldCBSProcessed {dOmega, coherent[],
        incoherent[]} where coherent/incoherent are vectors (one per time bin)
        of StokesMatrixProcessed {S0..S3}. Per-bin solid angle uses
        solid_angle_bin(i, dtheta, dphi).
(A radial-average helper for CBS is declared but commented out.)


12) Python API — the signatures that matter
===========================================
Ground truth: python/python_module.cpp. Highlights:

Phase functions:
  UniformPhaseFunction()
  RayleighPhaseFunction(nDiv, minVal, maxVal)
  HenyeyGreensteinPhaseFunction(g)
  RayleighDebyePhaseFunction(wavelength, radius, nDiv, minVal, maxVal)
  RayleighDebyeEMCPhaseFunction(wavelength, radius, n_particle, n_medium, nDiv, minVal, maxVal)
  DrainePhaseFunction(g, a, nDiv, minVal, maxVal)
  MiePhaseFunction(wavelength, radius, n_particle, n_medium, nDiv, minVal, maxVal)
  (EMC/Mie also expose pdf, rho_phase_function, scattering_efficiency,
   scattering_cross_section, get_anisotropy_factor.)

Media (constructors do NOT set transport; configure explicitly):
  RGDMedium(phase_func, radius, n_particle, n_medium, wavelength)
  MieMedium(phase_func, radius, n_particle, n_medium, wavelength)
  medium.set_mean_free_path(mfp)              # drives free-path sampling
  medium.set_scattering_coefficient(mu_s)     # drives absorption split + estimator
  medium.set_absorption_coefficient(mu_a)
  medium.scattering_cross_section()           # sigma_s [mm²] from the phase function
        (0 unless the phase function is EMC/Mie; used by MixtureLayer to derive
         mu_s^(i) = n_i · sigma_s^(i)). The RGDMedium/MieMedium ctors keep the
         phase function alive (py::keep_alive) since it is stored as a raw pointer.

Sample and layers:
  Sample(n_medium=1.0)
  sample.add_layer(medium, z_min, z_max)               # HomogeneousLayer (1 species)
  sample.add_mixture_layer(species, number_densities,  # MixtureLayer (N species)
                           z_min, z_max)
        # species: list of ScatteringMedium; number_densities: list of n_i [1/mm³];
        # both contiguous with the current top; use float('inf') for the top z_max.
        # The species list is kept alive for the Sample's lifetime (keep_alive).
  Layer types are exposed as Layer (abstract base: z_min, z_max, contains,
  thickness, mu_attenuation/mu_scattering/mu_absorption), SampleLayer(Layer)
  (single-species; .medium), and MixtureLayer(Layer) (.species, .number_densities,
  .mu_s_i, .selection_cdf, .mu_s_total, .mu_a_total, .mu_t_total, .mfp_total).
  sample.layers / sample.get_layer(i) return the polymorphic `Layer` base.

Laser:
  Laser(m_state, n_state, wavelength, sigma, source_type)        # source_type: LaserSource.{Point,Uniform,Gaussian}
  laser.set_temporal_profile(profile, pulse_duration=0.0, repetition_rate=0.0, time_offset=0.0)
  TemporalProfile values: Delta, Gaussian, TopHat, ExponentialTime, PulseTrain, CW
  (pulse_duration / time_offset are in ps; converted to ns internally.)

Sensors:
  PhotonRecordSensor(z, absorb=True)
  PlanarFieldSensor(z, len_x, len_y, dx, dy, absorb=True, estimator=False)
  PlanarFluenceSensor(z, len_x, len_y, len_t, dx, dy, dt, absorb=True, estimator=False)
  FarFieldCBSSensor(theta_max, phi_max, len_t, d_theta, d_phi, d_t, estimator=False)
        NOTE: d_theta/d_phi/d_t are bin WIDTHS, not counts. theta_pp_max is a
        read/write attribute (estimator angular cap).
  StatisticsSensor(z, absorb=False) + set_*_histogram_bins / set_time_resolution
  SensorsGroup(); det = group.add_detector(SensorObj)   # returns the internal handle

Absorption:
  Absorption(radius, depth, d_r, d_z, d_t=0.0, t_max=0.0)

Simulation:
  cfg = SimConfig()  # default ctor only; set fields by attribute
  cfg.n_photons, cfg.sample, cfg.laser, cfg.detector (required);
  cfg.absorption, cfg.n_threads (0=hw), cfg.batch_size, cfg.seed,
  cfg.MAX_EVENTS, cfg.track_reverse_paths, cfg.show_progress,
  cfg.progress_interval_pct, cfg.pin_threads_to_cores
  run_simulation_parallel(cfg)   # releases the GIL

Post-processing: postprocess_farfield_cbs / postprocess_planar_fluence /
  postprocess_planar_field (see §11).

KEEP-ALIVE GOTCHA: SimConfig.sample/laser/detector/absorption are stored as raw
pointers (return_value_policy reference). Keep the Python objects alive for the
whole run or you get use-after-free / segfault.


13) Minimal correct CBS workflow
================================
  phase  = MiePhaseFunction(wl, r, n_p, n_med, nDiv, 0.0, np.pi)
  medium = MieMedium(phase, r, n_p, n_med, wl)
  medium.set_mean_free_path(mfp)
  medium.set_scattering_coefficient(1.0/mfp)
  medium.set_absorption_coefficient(mu_a)
  sample = Sample(n_med); sample.add_layer(medium, 0.0, float('inf'))
  laser  = Laser(m_state, n_state, wl, sigma, LaserSource.Point)
  group  = SensorsGroup()
  det    = group.add_detector(FarFieldCBSSensor(theta_max, phi_max, len_t,
                                                d_theta, d_phi, d_t, estimator=True))
  cfg = SimConfig()
  cfg.n_photons = N; cfg.sample = sample; cfg.laser = laser; cfg.detector = group
  cfg.track_reverse_paths = True; cfg.n_threads = 0
  run_simulation_parallel(cfg)
  res = postprocess_farfield_cbs(det, N)
  eta = np.array(res.coherent[0].S0) / np.array(res.incoherent[0].S0)


14) Typed persistence layer (Python)
====================================
The save/load layer is schema-driven (rewritten; the old hand-written if/elif is
gone):
  - schema.py : SENSOR_SCHEMAS maps each sensor type to its meta_attrs/data_attrs
    (single source of truth for what is persisted; PhotonRecordSensor uses a
    custom encoder). COMMON_META lists the shared base-Sensor attributes.
  - records.py: typed dataclasses — params (SimParams/RunParams/LaserParams/
    MediumParams/LayerParams) and per-sensor results (FarFieldCBSResult,
    PlanarFluenceResult, PlanarFieldResult, StatisticsResult,
    PhotonRecordsResult, AbsorptionResult, and the *ProcessedResult variants),
    plus RESULT_BUILDERS used by the loader.
    LayerParams is discriminated by a `kind` field: "homogeneous" stores a single
    `medium: MediumParams`; "mixture" stores `species: List[MediumParams]`,
    `number_densities` and the aggregate mu_*_total / mfp_total. Its JSON
    round-trip goes through _layer_to_dict/_layer_from_dict, and old files without
    a `kind` load as homogeneous.
  - manager.py:
      Experiment(name, base_dir="sim_results", h5_name="results.h5",
                 timestamped=True) — creates a (timestamped) dir + results.h5
        with groups params/, sensors/<name>/{meta,data}/, derived/, absorption/.
        Key methods: save_params(config, extra=, units=) (auto-captures SimConfig
        + Sample + Laser + media via capture_params, stores a typed JSON blob in
        /params attr params_json plus a flat scalar projection), save_sensors(
        {name: sensor}) / save_sensor, save_processed(name, processed, sensor=),
        save_absorption(absorption, n_photons, name), save_derived,
        log_script/log_readme.
      ResultsLoader(exp_dir, h5_name="results.h5") — .params (typed SimParams
        from params_json), .params_flat (flat dict), typed accessors
        far_field_cbs / planar_fluence / planar_field / statistics /
        photon_records, processed_cbs / processed_fluence / processed_field,
        absorption(...), plus raw sensor_data / derived / [path] indexing.
      capture_params — auto-captures every layer (homogeneous or mixture) via the
        _layer_params helper; save_params' flat scalar projection reports the top
        layer's `layer_kind` and, for a mixture, medium_kind="mixture" + n_species
        + the aggregate mu_s/mu_a/mu_t/mfp_total.
      derived_quantities(medium, volume_fraction) — central g / l_s / l* /
        theta_coherent / size_parameter / m_relative for a single species.
      derived_quantities_mixture(species, number_densities) — the mixture analogue
        (typed: Sequence[ScatteringMedium], Sequence[float] -> Dict[str, Any]);
        returns per-species mu_s^(i)/g_i plus mu_s_total, g_eff, l_s, l* and
        theta_coherent. Both are meant to be passed as extra= to save_params.
  - sweepmanager.py: SweepManager for parameter sweeps.
  To add a sensor to persistence: one SENSOR_SCHEMAS entry + a *Result dataclass
  + a RESULT_BUILDERS builder. Nothing in manager.py changes.

Dev note: editing python/luminis_mc/*.py does NOT affect `import luminis_mc`
unless you re-sync the venv copy (scikit-build installs a non-editable copy).
For Python-only changes, symlink build/_core*.so into python/luminis_mc/ and run
with PYTHONPATH=python; or reinstall.


15) Implementation gotchas / dead code
======================================
  1. set_mean_free_path is REQUIRED — sample_free_path uses mean_free_path, not
     mu_s. Forgetting it makes photons never move.
  2. FarFieldCBSSensor takes bin WIDTHS (d_theta/d_phi/d_t), not counts.
  3. SimConfig has only a default ctor; set fields by attribute. Keep
     sample/laser/detector/absorption Python objects alive (raw pointers).
  4. add_detector takes ownership; read results from the returned handle.
  5. CBS requires track_reverse_paths=True.
  6. A sensor is direct OR estimator, never both; the estimator is only valid
     at z=0 and is auto-disabled (with a warning) otherwise.
  7. PlanarFieldSensor / PlanarFluenceSensor process_estimation are NOT
     implemented (the bodies are commented out). Use direct mode for them.
  8. src/cbs.cpp is NOT in the CMake source list → DEAD CODE (won't even compile
     standalone). The live CBS logic is in detector.cpp (reverse_field).
  9. core.hpp is a fully commented-out legacy header (no build content).
 10. Photon.detected_pos and Photon.coherent_path_calculated are dead fields.
 11. No ProgressMonitor class is bound; use SimConfig.show_progress /
     progress_interval_pct (and on_progress in manager.py for a simple printer).
 12. old/ scripts are legacy and frequently use outdated constructor signatures
     (old positional MieMedium/RGDMedium/SimConfig args, bin counts instead of
     widths, removed sensor types). Prefer results/ for current patterns and
     validate every signature against python/python_module.cpp.
 13. MixtureLayer needs a NON-ZERO scattering_cross_section() per species, which
     only EMC/Mie phase functions provide — other phase functions return 0 and
     the ctor throws (mu_s_total ≤ 0). The mixture's mu_s comes from
     n_i · sigma_s^(i), NOT from set_scattering_coefficient; but each species'
     set_scattering_coefficient/set_absorption_coefficient still set the per-
     species albedo used for the aggregate mu_a. As always, set_mean_free_path on
     each species is irrelevant to the mixture (the layer samples from mfp_total).
 14. Sample is MOVE-ONLY (owns unique_ptr<Layer>). Its Python `layers` /
     get_layer return the abstract `Layer`; downcast via isinstance to MixtureLayer
     / SampleLayer to read species vs medium. The species list passed to
     add_mixture_layer (and each phase function passed to a medium ctor) is stored
     as raw pointers — keep the Python objects alive for the whole run.
