                       LUMINIS-MC — Descripción del Código
================================================================================

1. VISIÓN GENERAL
================================================================================

luminis-mc es una librería de simulación Monte Carlo de transporte de fotones
en medios turbios, escrita en C++ con bindings de Python a través de pybind11.
Está diseñada para simular fenómenos de dispersión múltiple de luz, incluyendo
polarización, coherencia y Coherent Backscattering (CBS).

El usuario interactúa desde Python: configura el medio, el láser, los detectores
y ejecuta la simulación. Todo el cómputo pesado (propagación del fotón, scattering,
detección) se ejecuta en C++ compilado, con soporte para paralelismo multi-hilo.
El resultado se recoge en Python como arrays NumPy para su análisis y visualización.

La capa Python proporciona además un sistema de persistencia basado en HDF5
(Experiment + ResultsLoader) y un orquestador de barridos paramétricos
(SweepManager) para gestionar campañas de simulación a gran escala.

Tecnologías clave:
  - C++23 como núcleo computacional
  - Fortran 77 para el solver de Mie (MIEV0)
  - pybind11 para los bindings Python ↔ C++
  - scikit-build-core + CMake como sistema de compilación
  - std::thread para paralelismo
  - HDF5 (h5py) para persistencia de resultados

3. PIPELINE DE COMPILACIÓN Y USO
================================================================================

Compilación:
  pip install .    (o pip install -e .)

  Internamente:
    1. scikit-build-core invoca CMake
    2. CMake compila:
       a) mie_fortran  → librería estática Fortran (MIEV0 solver)
       b) _core_c      → librería estática C++ (toda la lógica de simulación)
       c) _core        → módulo Python (.so) que enlaza _core_c + pybind11
    3. Se instala como paquete Python: luminis_mc

Uso desde Python:
    import luminis_mc as lmc

    phase = lmc.MiePhaseFunction(...)
    medium = lmc.MieMedium(...)
    sample = lmc.Sample(n_medium=1.33)
    sample.add_layer(medium, 0.0, float('inf'))
    laser = lmc.Laser(...)
    sensors = lmc.SensorsGroup()
    det = sensors.add_detector(lmc.FarFieldCBSSensor(...))

    config = lmc.SimConfig(n_photons=1e6, sample=sample, ...)
    config.n_threads = 8
    lmc.run_simulation_parallel(config)

    # Acceder a resultados
    S0_coh = np.array(det.S0_coh, copy=False)


4. ESTRUCTURA DE LA SIMULACIÓN (run_photon)
================================================================================

El motor de la simulación implementa el método Monte Carlo de transporte radiativo
con polarización vectorial (matrices de Jones / parámetros de Stokes).

4.1 Configuración (SimConfig)
─────────────────────────────
SimConfig agrupa todos los componentes necesarios:
  - n_photons:           número total de fotones a simular
  - n_threads:           número de hilos para ejecución paralela
  - seed:                semilla del RNG (por defecto, std::random_device{})
  - sample:              puntero al Sample (muestra por capas compartiendo n_medio)
  - laser:               puntero a la fuente láser
  - detector:            puntero al grupo de sensores
  - absorption:          (opcional) registro de absorción cilíndrico (r,z) con
                         resolución temporal opcional
  - track_reverse_paths: si true, calcula el path reverso para CBS
  - progress:            (opcional) ProgressMonitor para reportar avance

Constructores:
  - SimConfig(n_photons, sample, laser, detector, absorption, track_reverse_paths)
      → semilla auto-generada desde std::random_device.
  - SimConfig(seed, n_photons, sample, laser, detector, absorption, track_reverse_paths)
      → semilla explícita para reproducibilidad.

4.2 Ejecución paralela (run_simulation_parallel)
─────────────────────────────────────────────────
1. Se distribuyen los n_photons entre n_threads hilos.
2. Cada hilo recibe:
   - Un clon del SensorsGroup (thread-local, sin locks)
   - Un clon del Absorption (thread-local)
   - Un RNG con semilla derivada (mix_seed(base_seed, thread_id))
3. Cada hilo ejecuta run_photon() para cada uno de sus fotones.
4. Si hay un ProgressMonitor, cada hilo llama tick() tras cada fotón
   (operación atómica lock-free).
5. Al finalizar todos los hilos, se hace merge de los detectores locales
   al detector principal (merge_from), y lo mismo con Absorption.

4.3 Loop de propagación del fotón (run_photon)
───────────────────────────────────────────────
Para cada fotón:

  (a) Inicialización:
      - El láser emite un fotón: posición, dirección, polarización (CVec2),
        longitud de onda, número de onda k.
      - Se muestrea el tiempo de emisión (launch_time) según el perfil
        temporal del láser (ver sección 7).
      - Se inicializan las matrices de base local P_local (3x3) que representan
        el sistema de coordenadas local del fotón: filas = (m, n, s) donde
        m y n son los ejes de polarización y s es la dirección de propagación.
      - La velocidad del fotón se establece como v = light_speed_in_medium()
        del Sample.
      - El fotón se ubica en la capa inicial (current_layer = 0).
      - Para CBS: se guardan P0, r_0, polarización inicial, T = I.

  (b) Step + cruce de interfaces:
      1. Muestrea un camino libre desde el medio de la capa actual.
      2. Si el paso cruza una interfaz entre capas (via find_next_interface):
         - Trunca el paso en la interfaz.
         - Calcula la distancia óptica restante: d_opt = (step_restante) × μ_t_vieja.
         - Mueve el fotón justo tras la interfaz (nudge ε).
         - Detecta sensores en el sub-paso (record_hit).
         - Transita a la nueva capa, convierte d_opt a paso físico:
             step_nuevo = d_opt / μ_t_nueva.
         - No actualiza velocidad: todas las capas comparten el mismo n_medio.
         - Repite si el paso restante cruza otra interfaz.
      3. Avanza lo restante del paso en la capa final:
           pos += P_local[2] * step

  (c) Detección: SensorsGroup::record_hit() verifica si el fotón cruzó algún
      plano detector (z_layers). Si cruza, calcula el punto de intersección,
      la fase óptica, la dirección de cruce (Forward/Backward), y llama a
      process_hit() del sensor correspondiente tras verificar filtros (incluyendo
      el filtro de dirección de cruce).
      Si absorb_photons=true, el fotón muere.

  (d) Registro de absorción: Si hay un Absorption configurado, en cada paso
      de scattering se deposita el peso absorbido:
        d_weight = weight × (μ_a / μ_t)
      en el bin cilíndrico (r, z) correspondiente a la posición del fotón.
      Si la absorción es time-resolved, el bin temporal se determina por
      launch_time + opticalpath / velocity.

  (e) Verificación de frontera: Comprueba que el fotón siga dentro del Sample.
      Si sale (z < 0 o z ≥ z_top), el fotón muere (sin reflexión de Fresnel).

  (f) Estimadores: SensorsGroup::run_estimators() permite que sensores con
      estimator_enabled calculen contribuciones parciales sin que el fotón
      llegue físicamente al plano detector.

  (g) Scattering:
      - Muestrea ángulo polar θ (desde PhaseFunction via CDF inversa)
      - Calcula la matriz de scattering S(θ) del medio
      - Muestrea ángulo azimutal φ condicionado a la polarización actual
        (muestreo condicional que respeta la distribución de Mie completa)
      - Actualiza la base local: P_local = A(θ,φ) · P_local
      - Actualiza polarización usando la matriz de Jones:
          T = S · R(φ)
          E_new = T · E_old / sqrt(F)
        donde F es el factor de normalización que asegura consistencia
        con el peso estadístico.

  (h) Tracking CBS (si track_reverse_paths):
      - Guarda r_1 (posición del primer scattering), r_n (último)
      - Guarda first_scatter_layer, last_scatter_layer
      - Guarda bases P0, P1, Pn2, Pn1, Pn
      - Acumula la matriz de Jones intermedia T_mid:
        Para n >= 3, T_mid = J_{n-1} · J_{n-2} · ... · J_2
        El buffer J_prev se acumula paso a paso (pipeline de 2 etapas).
      - También se acumula matrix_T_raw (sin el factor de normalización F)
        para uso en estimadores.

  (i) Absorción parcial y Russian roulette:
      - El peso se reduce: w *= μ_s / μ_t (albedo)
      - Si w < 1e-4: ruleta rusa (10% chance de sobrevivir con w/0.1)
      - Si events > 1000: el fotón muere.


5. EL FOTÓN (Photon)
================================================================================

Campos principales:
  - pos, prev_pos:        posición actual y anterior
  - detected_pos:         posición donde fue detectado por última vez
  - P_local:              Matrix 3x3, base local (m=fila0, n=fila1, s=fila2)
  - polarization:         CVec2 (Em, En) — campo eléctrico en base local
  - weight:               peso estadístico del fotón
  - k:                    número de onda 2π/λ
  - wavelength_nm:        longitud de onda en nm
  - opticalpath:          camino óptico acumulado
  - events:               número de scatterings sufridos
  - alive:                false cuando el fotón ha sido terminado
  - penetration_depth:    profundidad máxima alcanzada en el medio
  - launch_time:          tiempo de emisión [ns] (muestreado del perfil temporal
                          del láser); usado para detección time-gated y registro
                          de absorción temporal.
  - velocity:             velocidad de fase en el medio [mm/ns] (c/n_medium);
                          establecida a partir de Sample::light_speed_in_medium().
  - current_layer:        índice de la capa actual del Sample en la que se
                          encuentra el fotón. Se actualiza al cruzar interfaces.
  - polarized:            cuando false, se ignora la polarización y se usa
                          transporte incoherente. Por defecto true.

Campos para CBS:
  - P0, P1, Pn2, Pn1, Pn: bases locales en puntos clave de la trayectoria
  - r_1, r_n:             posiciones del primer y último scattering
  - first_scatter_layer:  índice de capa del primer scattering
  - last_scatter_layer:   índice de capa del último scattering
  - matrix_T:             matriz de Jones intermedia acumulada (J_2...J_{n-1}),
                          normalizada por factor F en cada paso
  - matrix_T_buffer:      buffer temporal para el pipeline de acumulación
  - matrix_T_raw:         producto acumulado sin normalización F (para estimadores)
  - matrix_T_raw_buffer:  buffer temporal para matrix_T_raw
  - has_T_prev:           true una vez que al menos un evento ha sido committed
  - initial_polarization: polarización del fotón al ser emitido
  - polarization_reverse: campo eléctrico del path reverso (calculado por CBS)
  - coherent_path_calculated: flag, true una vez calculado el path reverso

Método get_stokes_parameters():
  Retorna {S0, S1, S2, S3} calculados como:
    S0 = |Em|² + |En|²
    S1 = |Em|² − |En|²
    S2 = 2·Re(Em·En*)
    S3 = −2·Im(Em·En*)
  Si polarized es false, retorna {1, 0, 0, 0}.

PhotonRecord (struct separado):
  Snapshot inmutable de un fotón al momento de detección. Almacena:
  - events, penetration_depth, opticalpath, weight, k
  - launch_time:  tiempo de emisión [ns]
  - arrival_time: tiempo de llegada al detector [ns]
                  = launch_time + opticalpath / velocity
  - position_first_scattering, position_last_scattering, position_detector
  - direction, m, n (base local en el momento de detección)
  - polarization_forward, polarization_reverse (Jones vectors)


6. EL MEDIO DISPERSIVO (ScatteringMedium)
================================================================================

ScatteringMedium es la clase base abstracta que describe las propiedades de
scattering y absorción de las partículas en suspensión. NO almacena el índice
de refracción del medio huésped (solvente); ese dato pertenece al Sample
(sección 6b). Esto refleja la física: en una muestra real las distintas capas
pueden tener diferentes partículas en suspensión, pero el solvente (y por tanto
su índice de refracción) es el mismo en todas.

Archivo: include/luminis/core/medium.hpp   |   src/medium.cpp

6.1 ScatteringMedium (base abstracta)
─────────────────────────────────────
Métodos virtuales puros:
  - sample_free_path(rng)      → longitud del paso libre [mm]
  - scattering_matrix(θ, φ, k) → CMatrix 2×2 de Jones [[S2, 0], [0, S1]]

Métodos con implementación por defecto (delegan a PhaseFunction):
  - sample_scattering_angle(rng) → θ (via CDF inversa de la fase)
  - sample_azimuthal_angle(rng)  → φ uniforme en [0, 2π)
  - sample_conditional_azimuthal_angle(rng, S, E, k, θ) → φ condicionado
    a la polarización actual (muestreo que respeta la distribución de Mie)

Propiedades:
  - mu_absorption   (μ_a [1/mm])  — coeficiente de absorción
  - mu_scattering   (μ_s [1/mm])  — coeficiente de scattering
  - mu_attenuation  (μ_t = μ_a + μ_s [1/mm]) — coeficiente de atenuación total
  - phase_function  (puntero no-owning a PhaseFunction)

6.2 RGDMedium (Rayleigh-Gans-Debye)
────────────────────────────────────
Hereda de ScatteringMedium. Válido para partículas pequeñas y ópticamente
suaves: parámetro de tamaño x = 2πr/λ ≪ 1, |m − 1| ≪ 1 y |m − 1| x ≪ 1.

  - Camino libre: exponencial, l = −l_mean · ln(U), con l_mean = 1/μ_s.
  - Matriz de scattering: fórmula analítica RGD:
      s2 = −i k³ (m−1) V F(θ,k,r) cos(θ) / (2π)
      s1 = −i k³ (m−1) V F(θ,k,r)         / (2π)
    donde V = (4π/3) r³ es el volumen de la partícula, F el form factor, y
    m = n_particle/n_medium el índice relativo.
  - Parámetros propios: radius, n_particle, n_medium (contraste para el form
    factor, NO el índice global del solvente).

6.3 MieMedium (Mie completo)
────────────────────────────
Hereda de ScatteringMedium. Sin restricción de tamaño de partícula.

  - Camino libre: exponencial (igual que RGDMedium).
  - Matriz de scattering: tablas precomputadas por el solver Fortran MIEV0.
    Al construirse, se evalúan S1(θ) y S2(θ) en una grilla uniforme de 1000
    ángulos en [0, π] y se almacenan en DataTable para look-up O(1) por
    interpolación. Esto evita llamar a MIEV0 en cada evento de scattering.
  - Parámetros propios: radius, n_particle, n_medium, wavelength,
    m = n_particle/n_medium (complejo en general).

6.4 Funciones de fase (PhaseFunction)
─────────────────────────────────────
Todas heredan de PhaseFunction y se muestrean por inversión de CDF numérica:

  - UniformPhaseFunction:             isotrópica
  - HenyeyGreensteinPhaseFunction:    parametrizada por g (asimetría)
  - RayleighPhaseFunction:            scattering Rayleigh
  - RayleighDebyePhaseFunction:       Rayleigh-Debye-Gans
  - RayleighDebyeEMCPhaseFunction:    Rayleigh-Debye con corrección EMC
  - DrainePhaseFunction:              Draine (parametrizada por g y α)
  - MiePhaseFunction:                 exacta Mie (via tablas precalculadas)

Todas usan SamplingTable internamente: se construye la CDF numérica de la PDF
de scattering y se muestrea por inversión (sample_theta(u)).

Método adicional en PhaseFunction:
  - get_anisotropy_factor(n_samples): estima el factor de anisotropía g
    mediante muestreo Monte Carlo.


6b. LA MUESTRA POR CAPAS (Sample)
================================================================================

Archivo: include/luminis/core/sample.hpp   |   src/sample.cpp

El Sample modela una muestra física formada por capas contiguas de medios
dispersivos apiladas a lo largo del eje z, como un pastel de capas. Todas las
capas comparten el mismo medio huésped (solvente) con índice de refracción
n_medium; lo que cambia entre capas son las propiedades de las partículas en
suspensión (coeficientes ópticos μ_a, μ_s, función de fase, etc.).

6b.1 Modelo físico y justificación
──────────────────────────────────
En un experimento de scattering múltiple, la cubeta contiene un solvente
(p. ej. agua, n ≈ 1.33) en el que se suspenden partículas. Se puede preparar
la muestra con capas que difieren en concentración, tamaño de partícula o
material, pero el solvente es siempre el mismo líquido.

Consecuencias físicas:
  1. El índice de refracción es uniforme en toda la muestra:
       n(z) = n_medium  ∀ z.
  2. La velocidad del fotón es constante:
       v = c / n_medium = 1 / n_medium  (en unidades naturales c=1).
  3. No hay reflexión ni refracción de Fresnel en las interfaces entre capas,
     porque el contraste de índice es Δn = 0. El fotón cruza las interfaces
     libremente; solo cambian las propiedades de scattering/absorción.
  4. Al cruzar una interfaz, la dirección del fotón no cambia, pero el camino
     libre restante debe re-escalarse porque cambia el coeficiente de
     atenuación μ_t (ver sección 6b.3).

6b.2 Estructura de datos
────────────────────────

  SampleLayer (una capa individual):
    - medium:  const ScatteringMedium*  (puntero no-owning al medio dispersivo)
    - z_min:   frontera inferior de la capa [mm] (inclusiva)
    - z_max:   frontera superior de la capa [mm] (exclusiva, puede ser +∞)
    - contains(z)  → true si z_min ≤ z < z_max
    - thickness()  → z_max − z_min (puede ser +∞)

  Sample (colección ordenada de capas):
    - layers:           vector<SampleLayer>, ordenadas por z_min
    - interfaces:       vector<double>, coordenadas z de las fronteras internas
                        (excluye z=0 y z_max de la última capa)
    - refractive_index: índice de refracción del solvente (n_medium)
    - light_speed:      velocidad del fotón = 1 / n_medium

    Constructor:
      Sample(n_medium = 1.0)  → inicializa refractive_index y light_speed

    Métodos principales:
      - add_layer(medium, z_min, z_max)
          Agrega una capa al tope de la muestra. Reglas:
          · La primera capa debe empezar en z = 0.
          · Cada capa subsiguiente debe empezar exactamente donde termina la
            anterior (contigüidad estricta, sin huecos ni solapamientos).
          · z_max > z_min. La última capa puede tener z_max = +∞ (semi-infinita).
          · medium != nullptr.
      - get_layer(index)        → referencia a la capa por índice
      - get_layer_at(z)         → puntero a la capa que contiene z (búsqueda
                                  binaria O(log N) sobre el vector interfaces)
      - get_layer_index_at(z)   → índice de la capa que contiene z
      - find_next_interface(z_current, z_target)
          Busca la primera interfaz interna entre z_current y z_target.
          Retorna std::optional<double> con la coordenada z de la interfaz,
          o std::nullopt si no hay interfaz entre ambos puntos.
          Funciona tanto para movimiento ascendente (z_target > z_current)
          como descendente (z_target < z_current).
      - is_inside(position)     → true si la posición 3D está dentro del Sample
                                  (z ≥ 0 y z < z_top, sin NaN/Inf en x,y,z)
      - z_top()                 → z_max de la última capa (puede ser +∞)
      - light_speed_in_medium() → velocidad del fotón (= 1/n_medium)

6b.3 Algoritmo de cruce de interfaces (en run_photon)
─────────────────────────────────────────────────────
Cuando un fotón se mueve un paso libre (step) dentro de una capa, puede
cruzar una o varias interfaces antes de consumir toda la distancia óptica
muestreada. El algoritmo es iterativo:

  ENTRADA: step (distancia física muestreada), dirección (dir_x, dir_y, dir_z)

  1. Calcular z candidato: new_z = pos.z + dir_z * step
  2. Buscar la próxima interfaz: iface = find_next_interface(pos.z, new_z)
  3. Mientras exista una interfaz entre pos.z y new_z:
     a. Parametrizar la intersección: t = (z_iface − pos.z) / (dir_z · step)
        Si |dir_z · step| < ε o t ∉ [0,1], salir del loop (guard numérico).
     b. Distancia al borde: step_to_iface = step · t
     c. Distancia óptica restante: d_opt = (step − step_to_iface) × μ_t_vieja
        donde μ_t_vieja es el mu_attenuation de la capa actual.
     d. Mover el fotón a la interfaz:
          pos += dir · step_to_iface
          pos.z = z_iface ± ε   (nudge para aterrizar en la capa correcta)
     e. Verificar sensores en el sub-paso (record_hit).
     f. Determinar la nueva capa: new_layer_idx = get_layer_index_at(pos.z)
        Si new_layer_idx ≥ size(), el fotón salió del Sample → muere.
     g. Actualizar current_layer del fotón.
     h. Convertir distancia óptica a física en la nueva capa:
          step = d_opt / μ_t_nueva
        Esto preserva el número de caminos libres ópticos: la porción ya
        viajada consumió τ₁ = step_to_iface × μ_t_vieja, y el total muestreado
        era τ = step_original × μ_t_vieja. La porción restante τ₂ = τ − τ₁ se
        traduce a distancia física en el medio nuevo como τ₂ / μ_t_nueva.
     i. Recalcular new_z y buscar la siguiente interfaz (repite desde 3).

  4. Completar el paso restante en la capa final: pos += dir · step.

  Justificación del re-escaleo de distancia óptica:
    En Monte Carlo, el camino libre se muestrea como τ = −ln(U) (variable
    exponencial). Este τ es adimensional y representa la distancia óptica.
    La distancia física correspondiente es l = τ / μ_t. Al cambiar de capa,
    μ_t cambia, pero τ restante no: simplemente se convierte a la nueva
    longitud física l' = τ_restante / μ_t_nueva. Esto es equivalente a
    muestrear un nuevo camino libre condicionado a no haber interactuado
    en la primera capa — la "propiedad sin memoria" de la distribución
    exponencial.

  Nota sobre velocidad: como n_medium es constante, la velocidad del fotón
  no cambia al cruzar interfaces. No se aplican condiciones de Fresnel.

6b.4 Ejemplo de uso desde Python
────────────────────────────────

    # Funciones de fase para cada capa
    phase1 = lmc.MiePhaseFunction(radius=0.5e-3, n_p=1.59, n_m=1.33, wl=0.633e-3)
    phase2 = lmc.HenyeyGreensteinPhaseFunction(g=0.9)

7. EL LÁSER (Laser)
================================================================================

Archivo: include/luminis/core/laser.hpp   |   src/laser.cpp

Define la fuente de fotones que inicia cada trayectoria Monte Carlo.

7.1 Parámetros espaciales
────────────────────────
  - position:       origen de emisión (por defecto z = ε > 0, justo dentro
                    de la muestra)
  - direction:      dirección de propagación (por defecto +z)
  - local_m, local_n: base de polarización del láser (por defecto x, y)
  - polarization:   CVec2 (componentes m y n del campo eléctrico)
  - wavelength:     longitud de onda → determina k = 2π·n_medio / λ
  - sigma:          parámetro de dispersión espacial
  - source_type:    distribución espacial del haz:
      · LaserSource::Point    — fuente puntual (x=0, y=0)
      · LaserSource::Uniform  — disco uniforme de radio sigma
      · LaserSource::Gaussian — perfil gaussiano con desviación estándar sigma

7.2 Perfil temporal (Timed Laser)
─────────────────────────────────
El láser puede emitir fotones con distribución temporal configurable.
Cada fotón recibe un launch_time muestreado según el perfil elegido.
Esto permite simulaciones time-resolved cuando se combinan con sensores
que registran tiempo (PlanarFluenceSensor, FarFieldCBSSensor, Absorption).

Perfiles temporales (enum TemporalProfile):

  - Delta:        todos los fotones se emiten en t = t_offset.
                  Pulso infinitamente corto (función δ).

  - Gaussian:     distribución gaussiana centrada en t_offset con
                  FWHM = pulse_duration.
                  σ = pulse_duration × FWHM_TO_SIGMA, donde
                  FWHM_TO_SIGMA = 1 / (2√(2·ln2)) ≈ 0.4247.
                  t = Normal(t_offset, σ).

  - TopHat:       distribución uniforme en
                  [t_offset − pulse_duration/2, t_offset + pulse_duration/2].
                  Pulso rectangular de duración pulse_duration.

  - Exponential:  decaimiento exponencial con constante de tiempo pulse_duration.
                  t = t_offset − ln(U) × pulse_duration.
                  Modela un pulso con rise instantáneo y tail exponencial.

  - PulseTrain:   tren de pulsos equivalentes, uniformemente distribuidos
                  en una ventana temporal igual al período de repetición.
                  t = t_offset + U × (1/repetition_rate).
                  Parámetro: repetition_rate en Hz.

  - CW:           emisión continua (continuous wave) durante una ventana
                  temporal de duración pulse_duration.
                  t = t_offset + U × pulse_duration.

Parámetros de configuración temporal:
  - pulse_duration:  duración del pulso [ps]. Usado por Gaussian, TopHat,
                     Exponential, CW.
  - repetition_rate: tasa de repetición [Hz]. Usado por PulseTrain.
  - time_offset:     desplazamiento temporal [ps]. Se aplica a todos los perfiles
                     como un offset constante.

Conversión de unidades:
  Internamente, pulse_duration y time_offset se convierten de ps a ns
  (factor 1e-3) antes de asignar launch_time al fotón. Esto es consistente
  con la convención de que el camino óptico y la velocidad del fotón están
  en mm y mm/ns respectivamente.

Método set_temporal_profile(profile, pulse_duration, repetition_rate, time_offset):
  Configura el perfil temporal del láser. Si no se llama, el perfil por defecto
  es Delta con offset cero.

emit_photon(rng) → crea un Photon con posición (según source_type), dirección,
base local, polarización y launch_time (según temporal_profile) configurados.


8. LOS DETECTORES (Sensor y subclases)
================================================================================

8.1 Arquitectura general
────────────────────────
Todos los detectores heredan de la clase base abstracta Sensor:

  Sensor (base)
  ├── PhotonRecordSensor      — Graba cada fotón individualmente
  ├── PlanarFieldSensor        — Campo eléctrico complejo en plano (x,y)
  ├── PlanarFluenceSensor      — Parámetros de Stokes en plano (x,y,t)
  ├── PlanarCBSSensor          — CBS en plano espacial (en desarrollo)
  ├── FarFieldFluenceSensor    — Parámetros de Stokes en campo lejano (θ,φ)
  ├── FarFieldCBSSensor        — CBS en campo lejano (θ,φ,t) ← PRINCIPAL CBS
  └── StatisticsSensor         — Histogramas estadísticos

Cada Sensor tiene:
  - id (entero único asignado por SensorsGroup)
  - origin (posición z del plano detector)
  - normal (siempre ẑ)
  - backward_normal (siempre −ẑ)
  - m_polarization, n_polarization (base de referencia: +x, +y)
  - hits (contador de fotones procesados)
  - Filtros opcionales:
    · Theta: rango de ángulo polar permitido [theta_min, theta_max]
    · Phi: rango de ángulo azimutal permitido [phi_min, phi_max]
    · Posición: ventana rectangular (x_min, x_max, y_min, y_max)
    · Dirección de cruce: Forward (z+), Backward (z−), o Both
  - absorb_photons: si true, el fotón muere al ser detectado
  - estimator_enabled: si true, se llama process_estimation() en cada scattering

Filtro de dirección de cruce (CrossingDirection):
  Nuevo filtro que permite seleccionar fotones según la dirección en que
  cruzan el plano detector. Valores posibles:
    - CrossingDirection::Forward  — solo fotones viajando hacia z+
    - CrossingDirection::Backward — solo fotones viajando hacia z−
    - CrossingDirection::Both     — ambas direcciones (equivalente a desactivado)
  Se configura con set_direction_limit(direction).
  El InteractionInfo ahora incluye crossing_direction para que los sensores
  conozcan la dirección del cruce.

Métodos virtuales que cada sensor implementa:
  - process_hit():        se llama cuando el fotón cruza el plano detector
  - process_estimation(): se llama después de cada scattering (estimadores)
  - clone():              crea una copia vacía (para paralelismo)
  - merge_from():         combina resultados de otro sensor del mismo tipo

SensorsGroup:
  - Contenedor de múltiples sensores.
  - z_layers: mapa ordenado {z → [sensores en ese z]}. Permite buscar
    eficientemente qué sensores cruza un fotón verificando solo las capas
    entre z_min y z_max del paso actual. Usa DoubleComparator con tolerancia
    1e-9 para evitar duplicados por imprecisión floating-point.
  - Método add_detector(): clona el sensor y devuelve un puntero a la copia
    interna. Esto permite manipular el sensor devuelto directamente desde
    Python y leer resultados tras la simulación.
  - record_hit(): para cada capa z cruzada, calcula la intersección exacta,
    determina la CrossingDirection, verifica filtros (check_conditions), y
    llama process_hit().
  - run_estimators(): llama process_estimation() en todos los estimadores.

8.2 PhotonRecordSensor
──────────────────────
El sensor más simple. Almacena un vector de PhotonRecord.
Cada proceso de hit crea un snapshot completo del fotón: posición, dirección,
peso, polarización forward y reverse, camino óptico, launch_time, arrival_time,
etc.

Útil para: post-procesamiento personalizado en Python, cálculos de speckle,
análisis detallado por fotón individual.

8.3 PlanarFieldSensor
─────────────────────
Acumula el campo eléctrico complejo en una grilla 2D (N_x × N_y).
Tiene dos campos: Ex y Ey (CMatrix).

process_hit():
  - Proyecta la polarización local (Em, En) a componentes de laboratorio (x, y)
    usando la base P_local del fotón:
      Ex += (Em·P[0,0] + En·P[1,0]) · phase · √weight
      Ey += (Em·P[0,1] + En·P[1,1]) · phase · √weight
  - La fase es exp(i·k·optical_path) y √weight mantiene la coherencia
    estadística (la interferencia se produce al sumar amplitudes).

process_estimation():
  - Calcula la contribución del estimador: scatter virtual hacia el detector,
    incluyendo rotación de base, scattering matrix, atenuación exponencial,
    y proyección de polarización.

Útil para: patrones de speckle, interferencia coherente en campo cercano.

8.4 PlanarFluenceSensor (con resolución temporal)
──────────────────────────────────────────────────
Acumula los cuatro parámetros de Stokes (S0, S1, S2, S3) en una grilla
espacial 2D, opcionalmente resuelta en tiempo (S0_t[t_idx](x, y)).

Resolución temporal:
  - Si dt > 0: datos almacenados como S*_t[t_idx](x_idx, y_idx) con
    N_t = ceil(len_t / dt) bins temporales.
  - Si dt == 0: un solo bin temporal (time-integrated).
  El tiempo de llegada se calcula como: launch_time + opticalpath / velocity.

process_hit():
  - Calcula las componentes del campo en coordenadas del detector.
  - Calcula los Stokes: S0 = |Ex|² + |Ey|², S1 = |Ex|² - |Ey|²,
    S2 = 2·Re(Ex·Ey*), S3 = 2·Im(Ex·Ey*).
  - Deposita en el bin (x_idx, y_idx, t_idx) correspondiente.

process_estimation():
  - Similar al PlanarFieldSensor pero deposita parámetros de Stokes en vez
    de campo complejo.

Útil para: distribuciones espaciales de intensidad, resolución temporal,
imágenes de fluencia con polarimetría, propagación de pulsos.

8.5 FarFieldFluenceSensor
─────────────────────────
Acumula los Stokes en una grilla angular (N_theta × N_phi) en campo lejano.

process_hit():
  - Calcula θ = acos(-s_z) y φ = atan2(s_y, s_x) de la dirección del fotón.
  - Proyecta polarización a lab y deposita Stokes en el bin angular.

Útil para: distribuciones angulares de intensidad difusa, sin CBS.

8.6 StatisticsSensor
────────────────────
Histogramas configurables de propiedades del fotón al ser detectado:
  - events_histogram:  número de scatterings [0, max_events)
  - theta_histogram:   ángulo polar de salida [min_theta, max_theta)
  - phi_histogram:     ángulo azimutal [min_phi, max_phi)
  - depth_histogram:   profundidad de penetración [0, max_depth)
  - time_histogram:    tiempo de llegada [0, max_time)
  - weight_histogram:  peso estadístico [0, max_weight)

Se configura qué histogramas activar y sus rangos/bins con métodos
set_*_histogram_bins(). Histogramas no configurados permanecen vacíos
y no incurren en costo de ejecución.

Útil para: estadísticas de transporte, validación, distribuciones de
número de scattering, análisis time-of-flight.


9. DETECTORES CBS (Coherent Backscattering)
================================================================================

CBS es el fenómeno principal modelado por esta librería. La idea física es
que en backscattering, para cada trayectoria directa (forward) a través
del medio, existe una trayectoria inversa (reverse) que visita los mismos
scatterers en orden inverso. Estas dos trayectorias son recíprocas e
interfieren constructivamente en la dirección exacta de backscattering,
produciendo un pico de intensidad (enhancement factor ≈ 2).

9.1 PlanarCBSSensor (en desarrollo)
────────────────────────────────────
Diseñado para CBS en campo cercano (plano espacial). El código de process_hit()
está mayoritariamente comentado. La estructura almacena Stokes S0-S3 en una
grilla (N_x × N_y). No está funcional en la versión actual.

9.2 FarFieldCBSSensor ← DETECTOR CBS PRINCIPAL (con resolución temporal)
────────────────────────────────────────────────────────────────────────
Este es el detector central para simulaciones de CBS. Opera en el espacio
angular (campo lejano) con una grilla (N_theta × N_phi), opcionalmente
resuelta en tiempo.

Parámetros de construcción:
  FarFieldCBSSensor(theta_max, phi_max, t_max, n_theta, n_phi, n_t, estimator)
  - theta_max, phi_max: extensión angular máxima [rad]
  - t_max: ventana temporal total (0 para time-integrated)
  - n_theta, n_phi: número de bins angulares
  - n_t: número de bins temporales (ignorado si t_max=0)
  - estimator: habilitar estimador

Resolución temporal:
  - Si t_max > 0, dt = t_max / n_t. Los Stokes se almacenan como
    vector<Matrix> con N_t elementos (uno por bin temporal).
  - Si t_max == 0, N_t = 1, dt = 0 (time-integrated, un solo bin).
  - El tiempo de llegada se calcula igual que en PlanarFluenceSensor.

Parámetros de control del estimador (partial photon / next-event):
  - theta_pp_max: ángulo polar máximo para el estimador de fotón parcial.
                  Si < 0, usa theta_max. Permite limitar los bins sobre los
                  que itera el estimador para mejorar rendimiento.
  - theta_stride, phi_stride: subsampling para el estimador. Valores > 1
    saltan bins al iterar, reduciendo el costo computacional a costa de
    resolución angular en la contribución estimada.
  - _I_norm, _I_norm_k: caché mutable para la normalización angular
    (depende de k). Calculado una sola vez y reutilizado.

Almacena DOS conjuntos de Stokes (cada uno como vector<Matrix> de N_t bins):
  - S0_coh, S1_coh, S2_coh, S3_coh:     intensidad COHERENTE |E_f + E_r|²
  - S0_incoh, S1_incoh, S2_incoh, S3_incoh: intensidad INCOHERENTE |E_f|² + |E_r|²

El enhancement factor se obtiene como: η(θ) = I_coh(θ) / I_incoh(θ)
En θ=0 (exact backscattering), η → 2 para medios conservadores sin absorción.

process_hit(photon, info, medium):
  Requiere events >= 2 (CBS no existe para scattering simple).

  Paso 1: Cálculo del path reverso
    Llama a coherent_calculation(photon, medium) que calcula
    photon.polarization_reverse (ver sección 9.3).

  Paso 2: Ángulos far-field
    θ = acos(-s_out_z), φ = atan2(s_out_y, s_out_x)
    Se determina el bin (theta_idx, phi_idx).

  Paso 3: Bin temporal
    t_arrival = launch_time + opticalpath / velocity
    Se determina el bin t_idx = floor(t_arrival / dt).
    Si t_idx está fuera de rango, se descarta el fotón.

  Paso 4: Fase geométrica CBS
    La diferencia de fase entre path forward y reverse depende de la
    separación entre primer y último scatterer:
      q_b = k · (s_out + s_in)
      Δr = r_n - r_1
      path_phase = exp(i · q_b · Δr)
    En exact backscattering (s_out = -s_in), q_b = 0 y path_phase = 1,
    garantizando interferencia constructiva perfecta.

  Paso 5: Campos forward y reverse
    Forward:
      E_f = (Em_f, En_f) · P_local → (Efx, Efy) en lab
      con amplitud = phase · √weight  (info.phase = exp(i·k·optical_path))
    Reverse:
      E_r = (Em_r, En_r) · P_local → (Erx, Ery) en lab
      con amplitud adicional × path_phase

  Paso 6: Acumulación (en el bin temporal t_idx)
    Coherente:  E_total = E_f + E_r
      S0_coh[t_idx] += |Etx|² + |Ety|²
      S1_coh[t_idx] += |Etx|² - |Ety|²
      S2_coh[t_idx] += 2·Re(Etx·Ety*)
      S3_coh[t_idx] += 2·Im(Etx·Ety*)
    Incoherente: suma de intensidades individuales
      S0_incoh[t_idx] += (|Efx|² + |Efy|²) + (|Erx|² + |Ery|²)
      (análogamente para S1, S2, S3)

process_estimation():
  Implementa CBS con estimadores, iterando sobre bins angulares (con
  stride configurable) y calculando la contribución coherente e incoherente
  para cada dirección virtual. Usa coherent_estimation_partial() para
  calcular la polarización reversa con una base de salida hipotética.

9.3 Cálculo del path reverso: coherent_calculation()
─────────────────────────────────────────────────────
Esta función (en detector.cpp) calcula la polarización del path reverso
(photon.polarization_reverse) para un fotón que ha completado su
trayectoria forward.

La idea:
  Para una trayectoria con n scatterings en posiciones r_1...r_n,
  el path reverso recorre los mismos puntos en orden inverso: r_n...r_1.
  Se necesita construir la cadena de Jones del path reverso.

Notación del código:
  - s0: dirección incidente (del láser)
  - s1: dirección después del primer scattering
  - s_{n-1}: dirección antes del último scattering
  - s_n: dirección de salida (detectada)
  - P0, P1, Pn1, Pn: bases locales en esos puntos

Estructura del cálculo (3 bloques):

  Bloque A — Primer scattering reverso (en r_n):
    El path reverso "entra" con la dirección incidente s_0 y "sale" hacia
    -s_{n-1} (reverso del penúltimo tramo).
    - Se calcula θ_a = acos(s_0 · (-s_{n-1}))
    - Se obtiene S_a = scattering_matrix(θ_a)
    - Se construyen rotaciones para alinear las bases:
        R_n: base (m0,n0) → plano de scattering
    - scatter_event(S_a, R_n, E_initial) aplica la Jones matrix
    - R_n': plano de scattering → base (m_{n-1}, -n_{n-1})

  Bloque B — Scatterings intermedios (Q · T^T · Q):
    Para los scatterings 2...n-1, se usa el teorema de reciprocidad:
      La cadena de Jones del path reverso es Q · T^T · Q
    donde:
      T = J_2 · J_3 · ... · J_{n-1} (acumulada durante la propagación forward
          como photon.matrix_T)
      T^T = transpose (NO conjugado) de T
      Q = diag(1, -1) — refleja la componente n de polarización
    Esto evita re-calcular cada scattering individual del path medio.

  Bloque C — Último scattering reverso (en r_1):
    El path reverso "entra" desde -s_1 y "sale" hacia s_n (la dirección
    detectada del forward).
    - θ_b = acos((-s_1) · s_n)
    - S_b = scattering_matrix(θ_b)
    - Rotaciones análogas al bloque A
    - El resultado final se rota a la base (m_n, n_n) del punto de detección

El resultado es photon.polarization_reverse: CVec2 en la base local final
del fotón, listo para ser combinado con photon.polarization en el detector.

coherent_estimation_partial():
  Variante del cálculo reverso para estimadores. En lugar de usar la dirección
  de salida real del fotón, acepta una base de salida hipotética
  (last_scattering_P) para evaluar contribuciones virtuales hacia cualquier
  bin angular. También recibe una matriz T_mid intermedia, permitiendo su uso
  dentro de process_estimation().

Funciones auxiliares usadas (definidas en utils.hpp/cpp):
  - scatter_event(S, R, E):  E_out = S · R · E / √F  (normalizado)
  - rot2(m_to, n_to, m_from, n_from): CMatrix 2×2 de cambio de base
  - apply2(A, v): multiplica matriz 2×2 por vector 2×1
  - safe_unit(v, fallback): normaliza v, o usa fallback si es degenerado
  - clamp_pm1(x): clampea x a [-1, +1] para acos seguro
  - row_vec3(P, r): extrae la fila r de una Matrix 3x3 como Vec3

9.4 Post-procesamiento: postprocess_farfield_cbs() (con soporte temporal)
─────────────────────────────────────────────────────────────────────────
Función que normaliza los datos crudos del FarFieldCBSSensor.

Firma:
  postprocess_farfield_cbs(det, n_photons,
                           normalize_per_solid_angle=true,
                           normalize_per_photon=true,
                           eps=1e-30)

Parámetros de normalización:
  - normalize_per_solid_angle: si true, divide por el ángulo sólido del bin.
    Si false, dΩ se trata como 1 (útil para señal cruda).
  - normalize_per_photon: si true, divide por n_photons.
    Si false, deja la señal como suma acumulada.
  - eps: umbral mínimo para dΩ (evita divisiones por cero).

Para cada bin (i, j) y cada bin temporal t:
  dΩ = solid_angle_bin(i, dθ, dφ) = sin(θ_center) · dθ · dφ
  norm = (1/n_photons si normalize_per_photon) · (1/dΩ si normalize_per_solid_angle)
  S_coh_normalized[t](i,j) = S_coh_raw[t](i,j) · norm
  S_incoh_normalized[t](i,j) = S_incoh_raw[t](i,j) · norm

Retorna un FarFieldCBSProcessed que contiene:
  - dOmega: matriz (N_theta × N_phi) con el ángulo sólido de cada bin
  - coherent: vector<StokesMatrixProcessed> de longitud N_t, cada uno con
    matrices S0, S1, S2, S3 normalizadas
  - incoherent: vector<StokesMatrixProcessed> de longitud N_t

Structs de resultado:
  - StokesMatrixProcessed: S0, S1, S2, S3 como Matrix (N_theta × N_phi)
  - StokesRadialProcessed: S0, S1, S2, S3 como vector<double> (promedio radial)
  - FarFieldCBSProcessed: dOmega + vectores de coherent/incoherent
  - FarFieldCBSRadialProcessed: theta_center + coherent/incoherent radiales

Desde Python, se usa para calcular el enhancement factor:
  result = postprocess_farfield_cbs(det, n_photons)
  # Acceder al primer bin temporal (o único si time-integrated):
  enhancement = np.array(result.coherent[0].S0) / np.array(result.incoherent[0].S0)


10. REGISTRO DE ABSORCIÓN (Absorption)
================================================================================

Archivo: include/luminis/core/absortion.hpp   |   src/absortion.cpp

El módulo de absorción registra el peso depositado por cada fotón en una
grilla cilíndrica (r, z), opcionalmente resuelta en tiempo. Permite
visualizar cómo se distribuye la energía absorbida dentro del medio.

10.1 Modelo conceptual
─────────────────────
En Monte Carlo con absorción implícita (weight splitting), cada paso de
scattering reduce el peso del fotón por el factor de albedo:
    w_new = w_old × (μ_s / μ_t)
La diferencia d_weight = w_old − w_new = w_old × (μ_a / μ_t) es la fracción
de energía "depositada" (absorbida) en la posición actual. El Absorption
acumula estas contribuciones en bins cilíndricos.

10.2 Estructura de datos
───────────────────────
  Absorption:
    - radius:      extensión radial máxima de la grilla [mm]
    - depth:       extensión máxima en profundidad (z) [mm]
    - d_r:         ancho del bin radial [mm]
    - d_z:         ancho del bin en profundidad [mm]
    - d_t:         ancho del bin temporal [ns] (0 para time-integrated)
    - t_max:       ventana temporal total [ns] (ignorada cuando d_t = 0)
    - n_t:         número de bins temporales (1 cuando time-integrated)
    - time_slices: vector<Matrix>, cada elemento es una grilla n_r × n_z.
                   Longitud n_t.

  Resolución temporal:
    - d_t == 0  → un solo bin temporal (modo time-integrated)
    - d_t > 0   → n_t = ceil(t_max / d_t) bins temporales

10.3 Operaciones
───────────────
  Constructor: Absorption(radius, depth, d_r, d_z, d_t=0, t_max=0)
    Calcula n_r = floor(radius/d_r) + 1, n_z = floor(depth/d_z) + 1,
    n_t según d_t, y aloja todas las grillas con ceros.

  record_absorption(photon, d_weight):
    Determina el bin temporal: k = floor(arrival_time / d_t), donde
    arrival_time = launch_time + opticalpath / velocity.
    Calcula r = √(x²+y²), z = pos.z.
    Deposita d_weight en time_slices[k](i_r, i_z).
    Descarta si cualquier índice está fuera de rango.

  get_absorption_image(n_photons, time_index=0):
    Produce una imagen 2D simétrica (2·n_r × n_z) normalizada por:
      - volumen del vóxel cilíndrico: V = π·((r_out)² − (r_in)²) · d_z
      - número de fotones: n_photons
    La mitad izquierda es el espejo de la derecha, creando un mapa
    axialmente simétrico listo para visualización.

  clone() / merge_from(other):
    Patrón estándar para paralelismo (ver SensorsGroup). Cada hilo trabaja
    con un clon; al final se suman las grillas.

10.4 Ejemplo desde Python
─────────────────────────
  # Time-integrated absorption
  absorption = lmc.Absorption(radius=5.0, depth=10.0, d_r=0.05, d_z=0.05)

  # Time-resolved absorption
  absorption = lmc.Absorption(radius=5.0, depth=10.0, d_r=0.05, d_z=0.05,
                               d_t=0.1, t_max=10.0)

  config = lmc.SimConfig(n_photons=1e6, sample=sample, laser=laser,
                          detector=sensors, absorption=absorption)
  lmc.run_simulation_parallel(config)

  # Obtener imagen de absorción
  image = absorption.get_absorption_image(n_photons=1_000_000, time_index=0)
  plt.imshow(np.array(image), aspect='auto')


11. MONITOR DE PROGRESO (ProgressMonitor)
================================================================================

Archivo: include/luminis/log/progress.hpp

ProgressMonitor es un monitor lock-free que reporta el avance de la
simulación mediante un callback configurable. Diseñado para ejecución
multi-hilo sin contención.

11.1 Mecanismo
──────────────
  - Se configura con setup(total, callback, interval_pct).
  - Cada hilo de simulación llama tick() tras completar cada fotón.
  - tick() usa fetch_add atómico (sin lock) para incrementar el contador.
  - Cuando el contador cruza el siguiente umbral de reporte (calculado como
    total × interval_pct / 100), se dispara el callback exactamente una vez.
  - Un compare_exchange_weak loop garantiza que solo un hilo dispara cada
    umbral, evitando invocaciones concurrentes del callback.

11.2 Parámetros
──────────────
  - total:        número total de fotones a simular
  - callback:     función (photons_completed, total_photons) → void
  - interval_pct: intervalo de reporte como porcentaje (1–100, defecto 5%).
                  Con 5% y 1M fotones: ~20 invocaciones del callback.

11.3 Interfaz Python
───────────────────
El callback Python se envuelve con GIL acquisition ya que la simulación
corre con el GIL liberado:

  monitor = lmc.ProgressMonitor()
  monitor.setup(total=n_photons, callback=lmc.on_progress, interval_pct=5)
  config.progress = monitor

  # on_progress es un helper incluido en luminis_mc:
  def on_progress(done, total):
      pct = done * 100 // total
      print(f"\rSimulation progress: {pct}%", end="", flush=True)
      if done == total:
          print()

Métodos adicionales:
  - completed() → número de fotones completados hasta ahora
  - total()     → presupuesto total de fotones
  - is_enabled() → si el monitor está activo
  - disable()   → detiene futuros callbacks

13. MUESTREO DE CAMINO LIBRE MEDIO (Metropolis-Hastings)
================================================================================

Archivo: include/luminis/sample/meanfreepath.hpp   |   src/meanfreepath.cpp

Módulo auxiliar que implementa el algoritmo Metropolis-Hastings para muestrear
distribuciones de camino libre medio arbitrarias (no necesariamente exponenciales).

13.1 TargetDistribution (base abstracta)
────────────────────────────────────────
  - evaluate(x) → valor de la distribución objetivo en x.

Implementaciones concretas:
  - Exponential(lambda):  f(x) = λ·exp(−λ·x)
    Distribución exponencial estándar con tasa λ = 1/MFP.
  - HardSpheres(radius, density):  distribución de camino libre medio
    para un sistema de esferas duras con radio y densidad dados.

13.2 metropolis_hastings (sampler MCMC)
──────────────────────────────────────
  Constructor: metropolis_hastings(target_dist)
  Métodos:
    - sample(num_samples, initial_value, proposal_stddev, positive_support)
      Genera num_samples muestras MCMC desde la distribución objetivo.
    - accept_reject(current, target_current, proposal_stddev, positive_support)
      Un paso individual del algoritmo MH.
  Resultado: MCMC_samples (vector<double> accesible desde Python).


14. CAPA PYTHON: GESTIÓN DE EXPERIMENTOS
================================================================================

14.1 Experiment (manager.py)
───────────────────────────
Gestiona la persistencia completa de una simulación individual en un directorio
timestamped, almacenando todos los datos en un único archivo HDF5.

Layout HDF5:
    results.h5
    ├── params/           # escalares como attrs, arrays como datasets
    ├── sensors/
    │   └── <name>/
    │       ├── meta/     # tipo de sensor, dims, filtros (attrs)
    │       └── data/     # Stokes / campo / registros (datasets)
    ├── absorption/
    │   └── <name>/
    │       ├── meta/     # configuración de grilla (attrs)
    │       └── data/     # time_slice_N, time_slice_N_image (datasets)
    └── derived/          # arrays post-procesados listos para plotting

Directorio de salida:
  <base_dir>/<YYYY-MM-DD_HH-MM-SS>_<name>/
  ├── results.h5
  ├── script_snapshot.py     ← copia del script que lanzó el experimento
  └── README.md              ← notas opcionales

Constructor:
  Experiment(name, base_dir="sim_results", h5_name="results.h5", timestamped=True)
  - timestamped=True: prefijo con fecha/hora (directorio único por run).
  - timestamped=False: directorio fijo (sobrescribe resultados previos).

Métodos principales:

  log_params(units=None, **kwargs):
    Persiste parámetros de simulación en /params.
    - Escalares (str, int, float, bool, numpy scalars, Vec3, Vec2, CVec2,
      enums pybind11) → HDF5 attributes.
    - Arrays (numpy, list, matrices binding) → HDF5 datasets comprimidos (gzip).
    - Opcionalmente anota cada parámetro con su unidad.

  save_sensor(sensor, name):
    Persiste un sensor en /sensors/<name>/.
    - Guarda metadatos (tipo, dims, filtros) como attributes en /meta.
    - Guarda datos (Stokes, campos, registros) como datasets en /data.
    - Soporta TODOS los tipos de sensor:
      · PhotonRecordSensor: arrays de events, weight, depth, times,
        posiciones, polarizaciones.
      · PlanarFieldSensor: Ex, Ey como matrices complejas.
      · PlanarFluenceSensor: S0_t..S3_t como tensores (Nt × Nx × Ny).
      · PlanarCBSSensor: S0..S3 como matrices.
      · FarFieldFluenceSensor: S0..S3 como matrices.
      · FarFieldCBSSensor: S0_coh..S3_coh, S0_incoh..S3_incoh como arrays.
        También guarda parámetros de estimador (theta_pp_max, strides).
      · StatisticsSensor: todos los histogramas configurados.

  save_absorption(absorption, n_photons, name="absorption"):
    Persiste un Absorption recorder en /absorption/<name>/.
    - Guarda configuración de grilla en /meta.
    - Guarda cada time_slice como dataset.
    - Genera y guarda la imagen simétrica para cada slice.

  save_derived(key, arr, attrs=None):
    Almacena arrays post-procesados en /derived/<key>.
    Soporta claves jerárquicas con / (p.ej., "farfield_cbs/S0_coh").

  log_script(file_path):
    Copia el script de simulación al directorio del experimento.
    Registra la referencia en el HDF5.

  log_readme(text):
    Escribe contenido libre en README.md.

  update_log_readme(status, runtime_s, finished_at, error_message=None):
    Agrega bloque de estado al README.

  close():
    Flush + cierre del HDF5. También funciona como context manager:
      with Experiment("my_run") as exp:
          ...

Ejemplo completo:
    with lmc.Experiment("my_cbs_run") as exp:
        exp.log_script(__file__)
        exp.log_params(
            n_photons=1_000_000, mu_s=10.0, mfp=0.1,
            wavelength=633e-6, n_medium=1.33,
            units={"wavelength": "mm", "mfp": "mm"}
        )

        config = lmc.SimConfig(n_photons, sample, laser, sensors)
        lmc.run_simulation_parallel(config)

        exp.save_sensor(det, "far_field_cbs")
        exp.save_absorption(absorption, n_photons)

        result = lmc.postprocess_farfield_cbs(det, n_photons)
        exp.save_derived("enhancement/S0",
                         np.array(result.coherent[0].S0) / np.array(result.incoherent[0].S0))


14.2 ResultsLoader (manager.py)
──────────────────────────────
Accessor de solo lectura para archivos results.h5 previamente guardados.
Pre-carga todos los parámetros escalares en un diccionario.

Constructor:
  ResultsLoader(exp_dir, h5_name="results.h5")

Atributos:
  - params: Dict[str, Any] con todos los parámetros escalares y arrays.

Métodos:
  - sensor_data(sensor_name, dataset) → np.ndarray
    Carga /sensors/<name>/data/<dataset>.

  - sensor_meta(sensor_name) → Dict[str, Any]
    Retorna todos los attributes de /sensors/<name>/meta.

  - derived(key) → np.ndarray
    Carga /derived/<key>.

  - absorption_meta(name="absorption") → Dict[str, Any]
    Retorna attributes de /absorption/<name>/meta.

  - absorption_data(name, time_index=0) → np.ndarray
    Carga /absorption/<name>/data/time_slice_<idx>.

  - absorption_image(name, time_index=0) → np.ndarray
    Carga /absorption/<name>/data/time_slice_<idx>_image.

  - __getitem__(key) → np.ndarray
    Acceso directo por path HDF5:
      loader["sensors/far_field_cbs/data/S0_coh"]

Ejemplo:
    with lmc.ResultsLoader("/data/results/2026-02-19_my_run") as loader:
        S0_coh  = loader.sensor_data("far_field_cbs", "S0_coh")
        S0_inc  = loader.sensor_data("far_field_cbs", "S0_incoh")
        n_phot  = loader.params["n_photons"]
        image   = loader.absorption_image()


14.3 SweepManager (sweepmanager.py)
──────────────────────────────────
Orquestador de barridos paramétricos. Coordina múltiples Experiment bajo
un directorio raíz compartido, con tracking de cada run en un manifiesto
JSONL (newline-delimited JSON).

Directory layout:
    <base_dir>/<YYYY-MM-DD_HH-MM-SS>_<sweep_name>/
    ├── sweep_manifest.jsonl   # un registro JSON por run (started + finished)
    ├── script_snapshot.py     # copia del script maestro del barrido
    ├── README.md              # notas opcionales del barrido
    └── runs/
        ├── 0000_<run_name>/   # directorio de Experiment individual
        │   └── results.h5
        ├── 0001_<run_name>/
        ...

Constructor:
  SweepManager(sweep_name, base_dir, timestamped=True)
  - timestamped=True: directorio único con prefijo fecha/hora.
  - timestamped=False: directorio fijo (borra resultados previos).

Métodos:

  log_readme(text):
    Escribe README.md a nivel del barrido.

  snapshot_master_script(file_path):
    Copia el script maestro del barrido al directorio raíz.

  run(run_id, run_name, run_fn):
    Ejecuta un run individual del barrido.
    - run_id: índice entero (zero-based). Se usa para ordenar y nombrar
      el subdirectorio (zero-padded a 4 dígitos).
    - run_name: etiqueta legible.
    - run_fn: callable con firma run_fn(exp: Experiment) → None.
      Responsable de:
        1. Construir medio, láser, sensores, SimConfig.
        2. Ejecutar run_simulation_parallel().
        3. Guardar resultados via exp.save_sensor(), etc.

    Proceso interno:
      1. Crea subdirectorio runs/<run_id>_<run_name>/.
      2. Escribe registro "started" en sweep_manifest.jsonl.
      3. Crea un Experiment(name=..., timestamped=False) dentro de runs_dir.
      4. Llama run_fn(exp).
      5. Cierra el Experiment.
      6. Escribe registro "ok" o "failed" con runtime_s y finished_at.
      7. Si el run falló, re-lanza RuntimeError para que el caller decida.

Formato del manifiesto (JSONL):
  Cada línea es un JSON con: run_id, run_name, path, started_at, status,
  error (si hubo), runtime_s, finished_at.

Ejemplo:
    sweep = lmc.SweepManager("mfp_sweep", base_dir="/data/results")
    sweep.snapshot_master_script(__file__)
    sweep.log_readme("Sweep over mean free path values")

    for i, mfp in enumerate([0.5, 1.0, 2.0, 4.0]):
        def run_fn(exp, mfp=mfp):
            medium = lmc.MieMedium(mu_a=0, mu_s=1/mfp, phase=phase,
                                    mfp=mfp, radius=r, n_particle=np,
                                    n_medium=nm, wavelength=wl)
            sample = lmc.Sample(n_medium=nm)
            sample.add_layer(medium, 0.0, float('inf'))
            config = lmc.SimConfig(n_photons=1_000_000, sample=sample,
                                    laser=laser, detector=sensors,
                                    track_reverse_paths=True)
            config.n_threads = 8
            lmc.run_simulation_parallel(config)
            exp.save_sensor(det, "far_field_cbs")
            exp.log_params(mean_free_path=mfp, mu_s=1/mfp)

        sweep.run(run_id=i, run_name=f"mfp_{mfp}", run_fn=run_fn)

16. RESUMEN DE FLUJO CBS COMPLETO
================================================================================

1. Configurar:
   MiePhaseFunction → MieMedium → Sample(n_medium) → Laser →
   FarFieldCBSSensor(θmax, φmax, tmax, Nθ, Nφ, Nt)

2. Opcionalmente configurar perfil temporal del láser:
   laser.set_temporal_profile(lmc.TemporalProfile.Gaussian,
                               pulse_duration=100.0)

3. Opcionalmente configurar absorción:
   absorption = lmc.Absorption(radius, depth, d_r, d_z, d_t, t_max)

4. Opcionalmente configurar progreso:
   monitor = lmc.ProgressMonitor()
   monitor.setup(total=n_photons, callback=lmc.on_progress)

5. SimConfig(sample=sample, laser=laser, detector=sensors,
             absorption=absorption, track_reverse_paths=True)
   config.n_threads = 8
   config.progress = monitor

6. run_simulation_parallel():
   a. Cada fotón se emite con launch_time muestreado del perfil temporal.
   b. Se propaga acumulando T_mid y guardando bases P0..Pn.
   c. En cada paso, se registra absorción en el grid (r,z,t).
   d. Al cruzar z=0 hacia atrás (backscattering):
      - coherent_calculation() calcula E_reverse usando reciprocidad
      - FarFieldCBSSensor acumula |E_fwd + E_rev|² (coherente)
        y |E_fwd|² + |E_rev|² (incoherente) en bins (θ, φ, t)
   e. ProgressMonitor reporta avance periódicamente.

7. postprocess_farfield_cbs() normaliza por ángulo sólido y n_photons
   (por cada bin temporal).

8. Persistencia con Experiment:
   exp.save_sensor(det, "far_field_cbs")
   exp.save_absorption(absorption, n_photons)
   exp.save_derived("enhancement", enhancement_array)

9. Carga posterior con ResultsLoader:
   loader = ResultsLoader(exp_path)
   S0_coh = loader.sensor_data("far_field_cbs", "S0_coh")

10. En Python: η(θ) = S0_coh(θ) / S0_incoh(θ) → pico CBS en θ≈0
