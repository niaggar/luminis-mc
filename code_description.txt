LUMINIS-MC CODEBASE AND PHYSICS GUIDE
Updated: 2026-04-08

Purpose of this file
====================
This document is the onboarding reference for humans and AI agents working in
this repository. It explains:

1. What the code does physically.
2. How the C++ engine is structured.
3. How Python bindings expose the engine.
4. Which APIs are current (and which examples in tests are legacy/outdated).

If you only read one file before editing this project, read this one.


1) What this project simulates
==============================
Luminis-MC is a Monte Carlo simulator for light transport in scattering media,
with explicit polarization (Jones vectors) and coherent backscattering (CBS)
support.

Primary goal in this repository:
- Study coherent backscattering as interference between forward and reverse
  multiple-scattering paths.

Secondary capabilities:
- Time-resolved detectors.
- Layered samples.
- Absorption deposition maps.
- Statistics histograms.
- HDF5 persistence for large simulation campaigns.

4) Physical model and assumptions
=================================
This engine can be interpreted as a Monte Carlo solver of polarized transport
in random media, with coherent pair-interference added for CBS observables.

4.1 Governing transport picture
- At the scalar level, transport follows the radiative transfer equation form:

    dI(r, s_hat)/ds = -mu_t I(r, s_hat)
                      + mu_s integral_{4pi} p(s_hat' -> s_hat) I(r, s_hat') dOmega'
                      + S(r, s_hat)

  where mu_t = mu_a + mu_s.

- The Monte Carlo packet interpretation is:
  - free path is sampled stochastically,
  - direction is scattered using a phase-law sampler,
  - packet weight tracks absorption loss,
  - detector estimators accumulate observables.

4.2 Free-path and optical-depth sampling
- Free-path sampling is exponential in optical depth:

    tau = -ln(U),  U ~ Uniform(0,1)
    l   = tau / mu_t

- In code, RGDMedium and MieMedium implement sample_free_path with this
  exponential law using the configured mean_free_path (l_s = 1/mu_s).

4.3 Polarization state and moving local frame
- Each photon stores a local orthonormal frame P_local whose rows are:
  - row 0: m (transverse axis)
  - row 1: n (transverse axis)
  - row 2: s (propagation direction)

- Jones state is E = (E_m, E_n) in the (m,n) basis.
- At each scattering event, the basis rotates by A(theta, phi):

    P_local(new) = A(theta, phi) * P_local(old)

  with A implemented in src/simulation.cpp exactly as the frame update matrix.

4.4 Scattering operator and amplitude normalization
- The per-event Jones operator in code is:

    T = S(theta) * R(phi)

  where S(theta) is diagonal in the local scattering basis:

    S = [[S2, 0],
         [ 0, S1]]

- The photon Jones state update uses normalized amplitude transport:

    E_out = T E_in / sqrt(F)

- The normalization factor F used in src/simulation.cpp is:

    F = |E_m|^2 (|S2|^2 cos^2(phi) + |S1|^2 sin^2(phi))
      + |E_n|^2 (|S2|^2 sin^2(phi) + |S1|^2 cos^2(phi))
      + 2 Re(E_m E_n*) (|S2|^2 - |S1|^2) sin(phi) cos(phi)

  This keeps field amplitudes and packet-weight bookkeeping consistent.

4.5 Conditional azimuth sampling (polarization-aware)
- The azimuth phi is not always sampled uniformly.
- In phase.cpp, conditional sampling uses rejection sampling based on:

    F(phi) = a cos^2(phi) + b sin^2(phi) + c sin(phi) cos(phi)

  with

    a = |S2|^2 |E_m|^2 + |S1|^2 |E_n|^2
    b = |S1|^2 |E_m|^2 + |S2|^2 |E_n|^2
    c = 2 (|S2|^2 - |S1|^2) Re(E_m E_n*)

  and acceptance probability F(phi)/Fmax.

4.6 Medium models (RGD and Mie)
- RGDMedium (Rayleigh-Gans-Debye approximation):

    s2 = -i k^3 (m_rel - 1) V F_form(theta) cos(theta) / (2 pi)
    s1 = -i k^3 (m_rel - 1) V F_form(theta)            / (2 pi)

  where V = 4 pi r^3 / 3 and m_rel = n_particle / n_medium.

- MieMedium:
  - full Mie amplitudes S1(theta), S2(theta) from MIEV0,
  - precomputed on a theta grid and interpolated at runtime,
  - scattering efficiency Q_sca from mievinfo,
  - scattering cross section sigma_sca = Q_sca * pi r^2.

4.7 Layered sample and interface crossing physics
- All layers share one host refractive index n_medium via Sample.
- Therefore internal interfaces only change (mu_a, mu_s, phase model), not
  refractive index; no internal Fresnel reflection/refraction is modeled.

- The algorithm preserves remaining optical depth when crossing interfaces.
  If a sampled step of length l crosses an interface after l1 in old layer:

    tau_remaining = (l - l1) * mu_t(old)
    l_new         = tau_remaining / mu_t(new)

- Numerically, photon.z is nudged by +/-1e-12 after each crossing so layer
  lookup is robust against floating-point boundary ambiguity.

4.8 Absorption split and Russian roulette
- Weight splitting at each scattering event:

    d_weight = w * (mu_a / mu_t)
    w_new    = w * (mu_s / mu_t)

- d_weight is deposited into absorption grids (if enabled).
- Russian roulette is applied for low-weight packets (w < 1e-4):
  survive with p = 0.1 and renormalize w <- w/0.1, else terminate.

4.9 Time-of-flight and units used by detectors
- Arrival time used by time-resolved sensors:

    t_arrival = launch_time + opticalpath / velocity

- velocity is set from Sample::light_speed_in_medium() = 1 / n_medium
  in code units.
- Laser temporal parameters pulse_duration and time_offset are provided in ps
  but converted internally to ns before assignment to launch_time.

4.10 Stokes observables from Jones fields
- Detector-side Stokes conversions from field components (Ex, Ey):

    S0 = |Ex|^2 + |Ey|^2
    S1 = |Ex|^2 - |Ey|^2
    S2 = 2 Re(Ex Ey*)
    S3 = 2 Im(Ex Ey*)

- Photon::get_stokes_parameters uses a related convention in its local basis.
- Planar/FarField Stokes grids are Monte Carlo sums over packet contributions.

4.11 CBS-cone width relation used for interpretation
- For diffusion-like CBS analysis, a common reduced-variable scaling is
  q = k l* theta, with cone width inversely related to transport mean free
  path l*.
- A practical rule used in analysis scripts is:

    Delta q_FWHM ~ 0.73
    => theta_FWHM ~ 0.73 / (k l*)

  This is a postprocessing interpretation model, not a hard constraint inside
  the transport engine.


5) Core C++ types and responsibilities
======================================
Photon (include/luminis/core/photon.hpp)
- Live packet state during transport.
- Stores position, local frame P_local, Jones vector, weight, optical path,
  event count, current layer, launch time, velocity.
- For CBS: stores first/last scattering positions, local frame snapshots,
  accumulated transfer matrices, reverse-path output field.

Sample / SampleLayer (include/luminis/core/sample.hpp)
- Ordered contiguous z-layers with shared n_medium.
- Provides:
  - add_layer
  - get_layer_at / get_layer_index_at
  - find_next_interface
  - is_inside

ScatteringMedium hierarchy (include/luminis/core/medium.hpp)
- Base: ScatteringMedium
- Derived: RGDMedium, MieMedium
- IMPORTANT: constructors do not set optical coefficients automatically.
  You must set scattering/absorption explicitly from Python when needed.

Laser (include/luminis/core/laser.hpp)
- Source geometry (point, uniform disk, gaussian).
- Temporal profiles (delta/gaussian/tophat/exponential/pulse-train/cw).
- Emits Photon with launch_time sampled from selected temporal profile.

SimConfig (include/luminis/core/simulation.hpp)
- Run configuration for run_simulation_parallel.
- Includes n_photons, n_threads, seed, batch_size, show_progress,
  progress_interval_pct, track_reverse_paths, pin_threads_to_cores, and pointers
  to sample/laser/detector/optional absorption.


6) Simulation algorithm (run_photon)
====================================
High-level loop in src/simulation.cpp:

1. Resolve current layer and medium.
2. Sample free path.
3. Handle interface crossings (possibly multiple in one sampled step):
   - truncate at interface
   - preserve remaining optical depth
   - nudge photon by +/-1e-12 in z to land in next layer robustly
   - detect sensor intersections on each sub-step
4. Advance remaining step in final layer.
5. Record detector hits (SensorsGroup::record_hit).
6. Kill photon if outside sample.
7. Run estimators for estimator-enabled sensors.
8. Sample scattering angle theta and conditional azimuth phi.
9. Update local basis and Jones vector.
10. If track_reverse_paths enabled, update CBS path bookkeeping.
11. Apply absorption split and roulette.
12. Stop if killed or events exceed MAX_EVENTS.

Parallel driver (run_simulation_parallel):
- Clone-merge model:
  - each thread gets detector clone and optional absorption clone
  - each thread runs independent photons with per-thread RNG seed
  - final merge into shared detector/absorption objects
- Dynamic work scheduling by atomic next_photon and batch_size.


7) Sensor system and data products
==================================
Sensor base behavior
- Every sensor is a z-plane detector.
- SensorsGroup::record_hit computes exact crossing point and phase.
- Optional filters are conjunctive (all must pass):
  - theta range
  - phi range
  - x/y position window
  - crossing direction (Forward/Backward/Both)
  - event-count range (set_events_limit)

Sensor types:

1. PhotonRecordSensor
- Stores one PhotonRecord per hit.

2. PlanarFieldSensor
- Accumulates complex Ex/Ey on (x,y) grid.
- process_estimation is currently not implemented (commented/no-op path).

3. PlanarFluenceSensor
- Accumulates Stokes S0..S3 on (x,y) grid with optional time resolution.
- process_estimation is currently not implemented (commented/no-op path).

4. PlanarCBSSensor
- Marked WIP; not a stable production CBS path.

5. FarFieldFluenceSensor
- Accumulates Stokes on angular (theta,phi) bins.

6. FarFieldCBSSensor (main CBS detector)
- Accumulates coherent and incoherent Stokes fields, optionally time-resolved.
- Has estimator tuning fields: theta_pp_max, theta_stride, phi_stride.

7. StatisticsSensor
- Histogram collector for events/theta/phi/depth/time/weight.
- Supports temporal splitting with integrated bin semantics.


8) CBS implementation details
=============================
CBS in this codebase is computed in detector.cpp with a forward/reverse
pair-interference model.

8.1 Physical pair picture
- For a multiple-scattering path with at least two scattering events, a
  time-reversed partner exists that traverses the same scatterers in reverse
  order.
- Near exact backscattering these path pairs have nearly matched phase and can
  interfere constructively, producing enhancement above diffuse background.

8.2 Reverse-path reconstruction used in code
- Reverse field is computed by coherent_calculation(photon, sample).
- It uses stored path state from run_photon:
  - frame snapshots P0, P1, Pn2, Pn1, Pn
  - first and last scattering positions r_1 and r_n
  - accumulated transfer matrices (notably matrix_T_raw)

- The reconstruction is split into three blocks:
  1. first reverse scatter at r_n,
  2. middle segment via reciprocity transform,
  3. last reverse scatter at r_1.

- Middle segment reciprocity uses parity matrix Q and transposed transfer:

    E_mid_rev = Q * (T_mid)^T * Q * E_mid_fwd

  where T_mid is built from forward-path Jones products for interior events.

8.3 Geometric phase and interference term
- In FarFieldCBSSensor::process_hit, geometric pair phase is

    path_phase = exp(i * k * (s_out + s_in) dot (r_n - r_1))

- Forward and reverse lab-frame fields are then combined:

    E_tot = E_f + path_phase * E_r

- Coherent channel accumulates |E_tot|^2 Stokes.
- Incoherent channel accumulates |E_f|^2 + |E_r|^2 Stokes.

8.4 What the detector stores
- For each angular and temporal bin, FarFieldCBSSensor stores:
  - S0_coh..S3_coh
  - S0_incoh..S3_incoh

- Common observable derived in analysis:

    enhancement(theta, phi, t) = S0_coh / S0_incoh

  with exact-backscatter peak approaching 2 under ideal conservative,
  reciprocal conditions.

8.5 Temporal CBS accumulation convention
- If dt > 0, both coherent and incoherent contributions are written to:
  - bin 0 (always integrated), and
  - one explicit temporal bin t_idx = floor(t_arrival/dt) + 1.
- If dt == 0, only the integrated bin exists.

8.6 Validity conditions and limitations
- CBS coherent pair accumulation requires photon.events >= 2.
- track_reverse_paths should be true; otherwise reverse-path state is not
  consistently prepared for coherent pairing.
- The current implementation assumes reciprocal media and does not include
  non-reciprocal magneto-optic effects.


9) Time-resolved conventions (critical)
=======================================
There are two different time-binning patterns in the code. Do not mix them.

9.1 PlanarFluenceSensor and FarFieldCBSSensor
- dt == 0:
  - single bin only (N_t = 1), integrated signal.
- dt > 0:
  - N_t = ceil(len_t / dt) + 1
  - bin 0 is always integrated over all valid arrivals
  - bins 1..N_t-1 are explicit time windows.

9.2 StatisticsSensor temporal mode
- set_time_resolution(len_t, dt):
  - same integrated-bin convention as above for temporal hist arrays.

9.3 Absorption
- Different design from sensors:
  - if d_t > 0: n_t = ceil(t_max / d_t), no extra +1 integrated bin in
    time_slices
  - integrated data is stored separately in Absorption.total


10) Absorption recorder
=======================
Absorption (include/luminis/core/absortion.hpp, src/absortion.cpp):
- Cylindrical accumulation grid in (r,z), optional temporal slices.
- Stores both:
  - time_slices[k] (time-resolved)
  - total (always integrated)
- Exposed helpers:
  - get_absorption_image(n_photons, time_index)
  - get_total_image(n_photons)

Experiment.save_absorption in manager.py writes:
- /absorption/<name>/data/time_slice_i
- /absorption/<name>/data/time_slice_i_image
- /absorption/<name>/data/total
- /absorption/<name>/data/total_image


11) Post-processing functions
=============================
From src/results.cpp and python bindings:

- postprocess_planar_fluence
  - optional normalization by photon count and pixel area.

- postprocess_planar_field
  - optional normalization by photon count and pixel area.

13) Minimal correct Python workflow (current API)
=================================================
Use this pattern as a reliable starting point:

1. Build phase function:
   phase = MiePhaseFunction(wavelength, radius, n_particle, n_medium,
                            nDiv, 0.0, np.pi)

2. Build medium and set optical coefficients:
   medium = MieMedium(phase, radius, n_particle, n_medium, wavelength)
   medium.set_mean_free_path(mfp)
   medium.set_scattering_coefficient(1.0 / mfp)
   medium.set_absorption_coefficient(mu_a)

3. Build sample:
   sample = Sample(n_medium)
   sample.add_layer(medium, 0.0, float('inf'))

4. Build laser:
   laser = Laser(m_state, n_state, wavelength, sigma, LaserSource.Point)

5. Build sensors:
   sens = SensorsGroup()
   det = sens.add_detector(FarFieldCBSSensor(theta_max, phi_max,
                                             len_t, d_theta, d_phi, d_t,
                                             False))

6. Configure simulation:
   cfg = SimConfig()
   cfg.n_photons = N
   cfg.sample = sample
   cfg.laser = laser
   cfg.detector = sens
   cfg.track_reverse_paths = True
   cfg.n_threads = 0  # use hardware concurrency

7. Run:
   run_simulation_parallel(cfg)

8. Postprocess:
   result = postprocess_farfield_cbs(det, N)
   eta = np.array(result.coherent[0].S0) / np.array(result.incoherent[0].S0)


14) Important implementation gotchas
====================================
1. FarFieldCBSSensor constructor uses bin widths, not counts.
2. SimConfig has only default constructor in Python binding.
3. Many legacy scripts still pass old positional constructor arguments for
   MieMedium/RGDMedium/SimConfig.
4. No ProgressMonitor class is currently exported from bindings.
   Use SimConfig.show_progress and SimConfig.progress_interval_pct.
5. For CBS, if track_reverse_paths is false, reverse-path fields are not
   properly tracked for coherent calculations.
6. PlanarCBSSensor is work-in-progress; FarFieldCBSSensor is the stable CBS
   detector.
